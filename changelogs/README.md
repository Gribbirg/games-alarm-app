# Ченджлоги

Каждый PR добавляет **один** md-файл в `changelogs/unreleased/` с описанием того,
что изменилось **для пользователя**.

## Формат

- Имя файла: короткий слаг изменения, например `dark-theme.md`, `fix-alarm-sound.md`.
- Содержимое: один или несколько маркеров списка на русском языке, например:

  ```markdown
  - Добавлена тёмная тема в настройках.
  - Исправлен сбой при отключении будильника из уведомления.
  ```

- Пишите с точки зрения пользователя: что он увидит или заметит.
  Без деталей реализации, имён классов, рефакторингов и прочих технических подробностей.
- Если PR чисто технический и пользователь ничего не заметит, всё равно добавьте файл
  с одной строкой вида `- Технические улучшения стабильности и сборки.`

## Как это попадает в релиз

При запуске workflow `Release` все файлы из `changelogs/unreleased/` собираются
в новый раздел `CHANGELOG.md` с номером версии, публикуются как описание
GitHub-релиза и удаляются из `unreleased/`.

# Игра «Лабиринт» (`:feature:games:maze`)

Мини-игра для будильника SmartAlarm, **game id = 9**. Чтобы выключить
будильник, нужно провести фишку по лабиринту от входа до выхода.

## Правила

- Игрок стартует в **левой верхней** клетке (кружок), выход — **правая
  нижняя** клетка (скруглённый квадрат).
- Управление — четыре кнопки **↑ ↓ ← →** под полем лабиринта.
- Сквозь стены ходить нельзя: попытка шага в стену (или за границу поля)
  не двигает фишку, показывает подсказку «Там стена!» и **штрафуется**.
- Игра завершается победой, как только фишка оказалась на клетке выхода.

## Сложность

| Уровень | Размер лабиринта |
|---------|------------------|
| 1       | 7×7 клеток       |
| 2       | 9×9 клеток       |
| 3       | 11×11 клеток     |

Некорректное значение сложности не роняет игру — берётся размер 7×7
(`MazeGameLogic.sizeForDifficulty`).

## Генерация лабиринта

Алгоритм — **recursive backtracker** в итеративной форме (со стеком, без
рекурсии — нет риска переполнения стека на больших полях), см.
`Maze.generate`:

1. Создаётся решётка `N×N`, все стены на месте.
2. Из клетки (0,0) алгоритм случайно выбирает непосещённого соседа,
   убирает стену между клетками и переходит в него.
3. Когда непосещённых соседей нет — откат по стеку.
4. Работа заканчивается, когда стек пуст.

Результат — **идеальный лабиринт** (остовное дерево решётки):

- все клетки достижимы из любой;
- циклов нет, число проходов ровно `N*N − 1`;
- путь от входа к выходу существует и единственен.

Стены хранятся в двух общих массивах (горизонтальные и вертикальные),
поэтому стена между соседями по построению согласована с обеих сторон.
Внешние границы поля убрать нельзя (`removeWall` бросит исключение).
Случайность передаётся параметром `kotlin.random.Random` — при
фиксированном seed лабиринт воспроизводим (используется в тестах);
в игре берётся `Random.Default`, т.е. каждый запуск — новый лабиринт.

## Очки

Как в игре-образце «Арифметика»:

- каждая попытка пройти сквозь стену: **−10** очков;
- победа: **+ (600 − прошедшие_секунды) × сложность**.

Чем быстрее и аккуратнее пройден лабиринт и чем выше сложность, тем больше
очков. При очень долгой игре счёт может уйти в минус (поведение совпадает
с остальными играми).

## Архитектура

| Файл | Роль |
|------|------|
| `Maze.kt` | Чистая модель лабиринта (клетки, стены, `Direction`) и генератор `Maze.generate`. **Без зависимостей от Android.** |
| `MazeGameLogic.kt` | Чистая игровая логика: позиция игрока, `tryMove` → `MoveResult` (MOVED / BLOCKED / FINISHED), выход, размеры по сложности. **Без зависимостей от Android.** |
| `MazeView.kt` | Кастомная View: в `onDraw` рисует стены линиями, игрока кружком, выход квадратом. Цвета — из атрибутов темы (`colorOnBackground`, `colorPrimary`, `colorTertiary`) с контрастными fallback-значениями. Только читает модель. |
| `MazeGameViewModel.kt` | `AndroidViewModel` по образцу `CalcGameViewModel`: секундный таймер («м.сс»), очки, загрузка будильника из Room и его перезапуск, тонкая обёртка над `MazeGameLogic`. Лабиринт генерируется один раз — поворот экрана не сбрасывает игру. |
| `MazeGameFragment.kt` | Экран игры: контракт экрана игры (см. ниже), кнопки управления, навигация к результату. |
| `res/layout/fragment_maze_game.xml` | Разметка: таймер, `MazeView` (квадрат 1:1), крестовина из 4 кнопок, кнопка выключения музыки. |

## Контракт экрана игры

Аргументы фрагмента (ключи исторические, с пробелами): `alarm id` (Long),
`start time` (Long, 0 = проба), `test` (Boolean), `difficulty` (Int 1..3),
`music path` (String).

- `onResume`: если не `test` — снимается уведомление будильника
  (`NotificationManagerCompat`, id = `alarm id`).
- `onPause`: если не `test` и игра не пройдена — будильник ставится заново
  (`viewModel.startNewAlarm()`).
- Каждую чётную ненулевую минуту (секунды 00) при запуске будильником
  заново включается мелодия (`AlarmMediaPlayer.playAudio`) и появляется
  кнопка «Выключить музыку» (`AlarmMediaPlayer.stopAudio()`).
- Системная «назад»: в режиме `test` — возврат к выбору игр
  (`action_mazeGameFragment2_to_gameChoiceFragment`, popUpTo
  `alarmsFragment`); при запуске будильником — игнорируется (будильник
  нельзя «свернуть» кнопкой назад).
- Победа: в аргументы кладутся `time` (String), `score` (Int),
  `game id` = 9; навигация в `test` — 
  `action_mazeGameFragment2_to_gameResultFragment2`, иначе —
  `action_mazeGameFragment_to_gameResultFragment` +
  `viewModel.setPositiveResult()`.

Destination'ы и action'ы уже объявлены в `nav_graph.xml`,
`nav_game_graph.xml` (`:core:ui`) и `nav_games_demo_graph.xml` (демо).

## Тесты

`src/test/java/.../MazeTest.kt` и `MazeGameLogicTest.kt` (JUnit4, только
чистая логика, Android не требуется):

- размеры лабиринта по уровням сложности;
- идеальность сгенерированного лабиринта: BFS достигает все клетки, число
  проходов равно `клетки − 1` (для 7×7, 9×9, 11×11 и нескольких seed);
- согласованность стен между соседними клетками;
- внешние границы всегда на месте, их нельзя убрать;
- детерминизм: одинаковый seed → одинаковый лабиринт;
- движение: в стену — BLOCKED и позиция не меняется, в проход — MOVED,
  на клетке выхода — FINISHED; BFS-путь из старта всегда приводит к победе.

Запуск: `./gradlew :feature:games:maze:test`.

## Запуск в демо

Игра доступна в демо-приложении со всеми мини-играми (без будильника и
Firebase):

```bash
./gradlew :feature:games:demo:installDebug
```

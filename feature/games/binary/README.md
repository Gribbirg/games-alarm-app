# Мини-игра «Двоичный код»

Модуль `:feature:games:binary` (game id = **27**, пакет
`com.example.smartalarm.feature.games.binary`) — мини-игра будильника
SmartAlarm: чтобы выключить будильник, нужно переводить числа между
двоичной и десятичной системами счисления.

## Правила

- На экране показывается число в двоичной **или** десятичной записи —
  направление каждого вопроса выбирается случайно: «`101₂ = ?`»
  (варианты десятичные) или «`13 = ?₂`» (варианты двоичные).
- Под вопросом четыре кнопки-варианта с записями в «противоположной»
  системе; ровно один вариант верный. Двоичные записи всегда
  канонические — без ведущих нулей.
- Для победы нужно набрать N верных ответов. Ошибка **не сбрасывает**
  прогресс, но стоит **−10 очков**; после любого ответа (кроме
  победного) задаётся новый вопрос.
- Два подряд идущих вопроса никогда не загадывают одно и то же число.

## Сложности

| Сложность | Диапазон чисел    | Верных ответов для победы |
|-----------|-------------------|---------------------------|
| 1         | 1..15 (4 бита)    | 3                         |
| 2         | 1..63 (6 бит)     | 4                         |
| 3         | 1..255 (8 бит)    | 5                         |

## Дистракторы

Три неверных варианта подбираются правдоподобными — из чисел:

- **±1 и ±2** от загаданного (классическая ошибка «сбился на единицу»
  в младших разрядах);
- с **перевёрнутой битовой строкой** (например, 6 = `110` → `011` →
  3 = `11`; ведущие нули после переворота отбрасываются);
- с **одним изменённым битом** — инвертируется каждый существующий
  разряд и один разряд над старшим (110 → 111, 100, 010, 1110 …), такие
  записи визуально похожи на верную.

Кандидаты ограничиваются диапазоном сложности (и не равны загаданному
числу), из них случайно берутся три различных; если пул слишком мал
(например, для числа 1 на сложности 1), добирается случайными числами
диапазона. Все четыре варианта уникальны, порядок перемешивается.

## Конвертер

`BinaryNumbers` — чистый (без Android) конвертер:

- `toBinary(number)` — каноническая двоичная запись (без ведущих нулей)
  для чисел >= 1: 13 → `1101`.
- `fromBinary(binary)` — разбор канонической записи; пустая строка,
  посторонние символы и ведущие нули отвергаются
  (`IllegalArgumentException`).

## Очки

Как в игре «Калькулятор»:

- каждая ошибка: **−10**;
- при победе добавляется **(600 − прошло_секунд) × сложность**.

## Архитектура модуля

| Файл | Роль |
|------|------|
| `BinaryNumbers.kt` | Чистый конвертер двоичная ↔ десятичная запись. |
| `BinaryGame.kt` | Чистая логика партии: генерация вопросов и дистракторов, прогресс, исходы ответов (`BinaryGame`, `BinaryQuestion`, `QuestionDirection`, `AnswerResult`). Случайность инжектируется через `kotlin.random.Random` (детерминизм в тестах). |
| `BinaryGameViewModel.kt` | `AndroidViewModel`: таймер («м.сс»), очки, загрузка/перезапуск будильника (`AlarmDbRepository`, `AlarmCreateRepository`), тонкая обёртка над `BinaryGame` с публикацией `LiveData`. |
| `BinaryGameFragment.kt` | Экран игры: контракт экрана, привязка `LiveData` к UI, индексы систем счисления («₂») в тексте вопроса, обработка ответов, навигация. |
| `res/layout/fragment_binary_game.xml` | Разметка: таймер, прогресс, карточка вопроса (моноширинный шрифт), 4 кнопки-варианта, кнопка выключения музыки. |

Чистая логика (`BinaryNumbers.kt`, `BinaryGame.kt`) не импортирует
`android`/`androidx` и полностью покрывается JVM-тестами. В логике
вопрос хранится без индексов систем счисления — подписи «₂»/«= ?»
добавляет фрагмент при отображении.

## Контракт экрана игры

Соблюдается общий контракт (`.claude/rules/games.md`):

- аргументы: `alarm id` (Long), `start time` (Long, 0 = проба),
  `test` (Boolean), `difficulty` (Int 1..3), `music path` (String);
- `onResume`: снятие уведомления будильника (кроме `test`);
- `onPause`: перезапуск будильника через `viewModel.startNewAlarm()`,
  если игра не пройдена (кроме `test`);
- каждые две минуты бездействия — повторное включение мелодии
  (`AlarmMediaPlayer.playAudio`) и показ кнопки «Выключить музыку»;
- системная «назад»: в режиме `test` — возврат к выбору игр
  (`action_binaryGameFragment2_to_gameChoiceFragment`), при будильнике —
  ничего;
- победа: в аргументы кладутся `time` (String), `score` (Int),
  `game id` (Int = 27), затем переход к результату:
  `action_binaryGameFragment2_to_gameResultFragment2` (проба) или
  `action_binaryGameFragment_to_gameResultFragment` (будильник, плюс
  `viewModel.setPositiveResult()`).

## Тесты

`src/test/java/.../binary/` (JUnit 4, только чистая логика):

- `BinaryNumbersTest` — известные значения `toBinary`/`fromBinary`
  (1, 2, 5, 10, 13, 15, 63, 64, 128, 255), полный roundtrip
  `fromBinary(toBinary(n)) == n` для 1..255 с проверкой каноничности
  записи, отказ на 0/отрицательных числах и некорректных строках
  (пустая, `0`, `011`, `102`, `13`, ` 101`, `-1`, `abc`).
- `BinaryGameTest` — диапазоны и цели по сложностям (включая трактовку
  неизвестной сложности как максимальной), согласованность вопроса с
  направлением (перевод пересчитывается в тесте), уникальность четырёх
  вариантов и наличие верного, формат двоичных вариантов (только `0`/`1`
  без ведущих нулей), неповторение чисел подряд, детерминизм с
  одинаковым seed, прогресс/победа/ошибки, отказ на неверном индексе.

Запуск: `./gradlew :feature:games:binary:test`

## Запуск в демо

Игра добавлена в демо-приложение со всеми мини-играми (выбор сложности,
без будильника и Firebase):

```bash
./gradlew :feature:games:demo:installDebug
```

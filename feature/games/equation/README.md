# Игра «Уравнение» (`:feature:games:equation`)

Мини-игра для будильника SmartAlarm (**game id = 3**, пакет
`com.example.smartalarm.feature.games.equation`). Игроку показывается
равенство с одним пропущенным элементом и четыре кнопки-варианта; нужно
выбрать то, что стоит на месте `?`. Чтобы выключить будильник, надо решить
несколько уравнений.

## Правила

- На экране — уравнение с `?` (например `12 + ? = 20` или `12 ? 3 = 4`)
  и 4 варианта ответа. Ровно один вариант верный.
- Верный ответ увеличивает прогресс («Решено: N / M») и выдаёт следующее
  уравнение.
- Неверный ответ **не сбрасывает** накопленный прогресс, но снимает 10 очков
  и выдаёт **новое** уравнение — подобрать ответ перебором четырёх кнопок
  нельзя.
- Победа — когда решено нужное число уравнений; после этого игрок попадает
  на экран результата, и будильник больше не перезапускается.

## Сложности

| Сложность | Уравнений до победы | Вид задания | Параметры генерации |
|---|---|---|---|
| 1 | 3 | Пропущено **число** (любой операнд или результат) в одном действии `+ − × ÷` | `+`: 1..9 + 1..9; `−`: уменьшаемое 2..18, разность ≥ 1; `×`: 2..9 × 2..9; `÷`: делимое = делитель(2..9) × частное(2..9) — всегда нацело |
| 2 | 4 | Пропущен **оператор**: `12 ? 3 = 4` → `÷`; варианты — все четыре оператора | `+`/`−`: числа 10..99 (разность ≥ 1); `×`: 3..12 × 3..12; `÷`: делитель и частное 2..12. Гарантируется, что равенство выполняет **ровно один** оператор |
| 3 | 5 | Пропущено **число** в двухшаговом выражении `a ∘ b ∘ c = d`, операторы `+ − ×` | Операнды при `×` — 2..9, при `+`/`−` — 10..40 (третье число 2..30); двух умножений подряд не бывает; результат `d` в пределах 0..200. Приоритет стандартный: умножение раньше сложения/вычитания |

Отрицательные числа нигде не показываются (ни в уравнении, ни в вариантах),
деление всегда нацело.

### Варианты ответа

- Для числовых заданий: верный ответ + 3 дистрактора в пределах ±10 от него,
  все уникальные, не меньше 0 (для операндов — не меньше 1), и ни один
  дистрактор не удовлетворяет уравнению (проверяется подстановкой).
- Для задания с оператором (сложность 2): варианты — все четыре оператора
  `+ − × ÷` в случайном порядке.

## Очки

- Каждая ошибка: **−10** очков.
- При победе (как в игре «Арифметика»):
  `score += (600 − прошло_секунд) × сложность`.
  Итог может быть отрицательным, если решать очень долго или много ошибаться.

## Архитектура

Чистая логика отделена от Android и покрыта юнит-тестами; вся случайность
идёт через `kotlin.random.Random`, поэтому с фиксированным seed поведение
детерминировано.

| Файл | Роль |
|---|---|
| `Operator.kt` | Enum операторов `+ − × ÷`; `apply(a, b)` возвращает `null` при делении с остатком/на ноль. Без Android-зависимостей |
| `EquationTask.kt` | Модель задания: текст с `?`, верный ответ, 4 варианта. Без Android-зависимостей |
| `EquationGenerator.kt` | Генерация заданий по сложности: уравнение, ответ, дистракторы. Без Android-зависимостей |
| `EquationGame.kt` | Состояние партии: число раундов (3/4/5), прогресс, обработка ответа (`AnswerResult`: CORRECT / WRONG / WIN). Без Android-зависимостей |
| `EquationGameViewModel.kt` | `AndroidViewModel`: таймер «м.сс», очки, загрузка/перезапуск будильника (`AlarmDbRepository`, `AlarmCreateRepository`), LiveData задания и прогресса. Партия переживает поворот экрана |
| `EquationGameFragment.kt` | Экран игры: контракт экрана (см. ниже), клики по кнопкам, навигация к результату |
| `res/layout/fragment_equation_game.xml` | Разметка: таймер, прогресс, карточка с уравнением, 4 кнопки 2×2, кнопка выключения музыки |

## Контракт экрана игры

Реализован по образцу `CalcGameFragment` (см. `.claude/rules/games.md`):

- аргументы: `"alarm id"` (Long), `"start time"` (Long, 0 — проба),
  `"test"` (Boolean), `"difficulty"` (Int 1..3), `"music path"` (String);
- `onResume`: если не `test` — снимается уведомление будильника;
- `onPause`: если не `test` и игра не пройдена — будильник перезапускается;
- на каждой чётной ненулевой минуте заново включается мелодия
  (`AlarmMediaPlayer.playAudio`) и появляется кнопка её выключения;
- системная «назад»: в режиме `test` — возврат к выбору игр
  (`action_equationGameFragment2_to_gameChoiceFragment`), при реальном
  будильнике — игнорируется (выключить будильник можно только победой);
- победа: в аргументы кладутся `"time"` (String), `"score"` (Int),
  `"game id"` (Int = 3) и выполняется переход к результату
  (`action_equationGameFragment2_to_gameResultFragment2` в режиме `test`,
  иначе `action_equationGameFragment_to_gameResultFragment`).

## Тесты

`src/test/java/.../EquationGeneratorTest.kt`, `EquationGameTest.kt`
(JUnit 4, только чистая логика):

- подстановка верного ответа делает равенство истинным (все сложности,
  независимый вычислитель в тесте);
- 4 уникальных варианта, среди них верный; дистракторы уравнение
  **не** решают;
- для сложности 2 варианты — ровно четыре оператора;
- деление всегда нацело, отрицательных чисел нет;
- детерминизм генерации и партии при фиксированном seed;
- число раундов по сложности, прогресс, ошибки (не двигают прогресс,
  но и не сбрасывают его), победа и поведение после победы.

Запуск: `./gradlew :feature:games:equation:test`.

## Запуск в демо

Игра добавлена в демо-приложение со всеми мини-играми (выбор сложности,
без будильника и Firebase):

```bash
./gradlew :feature:games:demo:installDebug
```

На устройстве открыть «SmartAlarm Демо» → «Уравнение».

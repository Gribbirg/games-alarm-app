# Мини-игра «Продолжи ряд» (`:feature:games:sequence`)

Мини-игра будильника SmartAlarm, **game id = 6** (см. `ALL_GAMES` в `:core:data`).
Пакет: `com.example.smartalarm.feature.games.sequence`.

## Правила

На экране показывается числовой ряд из 4–5 членов, например `2, 5, 8, 11, ?`,
и четыре варианта ответа. Игрок должен выбрать число, продолжающее ряд.
Верный ответ открывает следующий ряд; для победы (и выключения будильника)
нужно решить несколько рядов подряд:

| Сложность | Рядов для победы | Доступные типы рядов |
|-----------|------------------|----------------------|
| 1 | 3 | арифметическая прогрессия |
| 2 | 4 | + геометрическая прогрессия, квадраты |
| 3 | 5 | + чередующиеся шаги, фибоначчи-подобные |

Неверный ответ не меняет ряд — можно пробовать снова, но каждая ошибка
снимает очки.

## Типы рядов и параметры генерации

Все параметры подобраны так, чтобы числа оставались небольшими (переполнение
`Int` невозможно) и неотрицательными:

- **Арифметическая прогрессия** — 4–5 членов, шаг 2..9, растущая или
  убывающая (стартовое число убывающей подбирается так, чтобы ряд и ответ
  были ≥ 0). Пример: `3, 10, 17, 24, ?` → 31.
- **Геометрическая прогрессия** — 4 члена, первый член 2..5, знаменатель
  2..3 (максимальный ответ 5·3⁴ = 405). Пример: `2, 6, 18, 54, ?` → 162.
- **Квадраты** — 4–5 последовательных квадратов, начиная с k = 1..7
  (максимальный ответ 12² = 144). Пример: `4, 9, 16, 25, ?` → 36.
- **Чередующиеся шаги** — 5 членов, попеременно +a и −b, где a = 4..9,
  b = 1..a−1, старт 1..10 (ряд в целом растёт и остаётся положительным).
  Пример: `3, 10, 8, 15, 13, ?` → 20.
- **Фибоначчи-подобный** — 5 членов, t₁ = 1..5, t₂ = t₁..9, дальше каждый
  член — сумма двух предыдущих (максимальный ответ 60).
  Пример: `2, 3, 5, 8, 13, ?` → 21.

### Варианты ответа (дистракторы)

К верному ответу добавляются три уникальных дистрактора, отражающих типичные
ошибки: «продолжил с последним шагом» (например, арифметическое продолжение
геометрического ряда), неверный шаг чередования, пропущенный квадрат,
удвоение последнего члена, промах на ±1/±2. Дистракторы никогда не совпадают
ни между собой, ни с верным ответом (страховочный добор значениями
`ответ ± Δ`). Четыре варианта перемешиваются.

## Очки

Как в игре «Арифметика»:

- каждая ошибка: **−10** очков;
- победа: **+ (600 − прошло_секунд) × сложность**.

## Архитектура

Чистая логика не имеет зависимостей от Android, вся случайность идёт через
параметр `kotlin.random.Random` (детерминизм с фиксированным seed):

| Файл | Роль |
|------|------|
| `SequenceTask.kt` | `SequenceType` (типы рядов) и `SequenceTask` — одно задание: члены ряда, верный ответ, 4 варианта, текст «2, 5, 8, 11, ?», проверка ответа. |
| `SequenceGenerator.kt` | Чистый генератор: типы по сложности (`typesFor`), число рядов (`roundsFor`), генерация ряда каждого типа и подбор дистракторов. |
| `SequenceGame.kt` | Чистое состояние партии: текущее задание, прогресс (`roundsSolved`/`totalRounds`), счётчик ошибок, `submitAnswer`, признак победы `isFinished`. |
| `SequenceGameViewModel.kt` | `AndroidViewModel`: секундный таймер («м.сс»), очки, загрузка будильника (`AlarmDbRepository`) и его перезапуск (`AlarmCreateRepository`), тонкая обёртка над `SequenceGame`. |
| `SequenceGameFragment.kt` | Экран игры: отрисовка ряда и кнопок, обработка ответов, контракт экрана игры (см. ниже). |
| `res/layout/fragment_sequence_game.xml` | Разметка: таймер, счётчик «Ряд N из M», карточка с рядом, 4 кнопки вариантов, кнопка выключения музыки. Цвета — через `?attr/...` темы. |

## Контракт экрана игры

Аргументы фрагмента (исторические ключи с пробелами): `alarm id` (Long),
`start time` (Long, 0 — проба), `test` (Boolean — запуск из выбора игр),
`difficulty` (Int 1..3), `music path` (String).

- `onResume` — снимает уведомление будильника (кроме `test`);
- `onPause` — перезапускает будильник через `viewModel.startNewAlarm()`,
  если игра не пройдена (кроме `test`);
- каждые две минуты бездействия заново включается мелодия
  (`AlarmMediaPlayer.playAudio`) и появляется кнопка её выключения;
- системная «назад»: в режиме `test` — возврат к выбору игр
  (`action_sequenceGameFragment2_to_gameChoiceFragment`), в режиме будильника —
  игнорируется (будильник нельзя обойти);
- при победе в аргументы кладутся `time` (String), `score` (Int),
  `game id` (Int = 6) и выполняется переход к результату:
  в `test` — `action_sequenceGameFragment2_to_gameResultFragment2`,
  в режиме будильника — `action_sequenceGameFragment_to_gameResultFragment`
  плюс `viewModel.setPositiveResult()`.

Destination'ы объявлены в обоих nav-графах `:core:ui` (`nav_graph.xml`,
`nav_game_graph.xml`) и в графе демо-приложения.

## Тесты

`src/test/java/...` (JUnit4, только чистая логика, без Android):

- `SequenceGeneratorTest` — каждый тип ряда следует своему закону и ответ
  корректен; варианты уникальны и содержат верный ровно один раз; типы по
  сложностям; детерминизм с seed; коэрция сложности.
- `SequenceGameTest` — прогресс по верным ответам, ошибки не двигают ряд,
  победа после всех рядов, игнорирование ответов после победы, детерминизм,
  число рядов по сложностям.

Запуск: `./gradlew :feature:games:sequence:test`.

## Запуск в демо-приложении

Игру можно запустить без будильника и Firebase через демо со всеми
мини-играми:

```bash
./gradlew :feature:games:demo:installDebug
```

В лаунчере демо выбрать «Продолжи ряд» и сложность.

# Игра «Наоборот» (`:feature:games:reverse`)

Мини-игра будильника SmartAlarm (game id = **28**). Игроку показывается
русское слово, написанное **задом наперёд** (например, «АКОРОС»), и четыре
варианта ответа; нужно выбрать исходное слово («сорока»). Чтобы выключить
будильник, надо дать несколько верных ответов подряд по счётчику прогресса.

## Правила

- На экране — перевёрнутое слово и 4 кнопки-варианта; ровно один вариант
  правильный.
- Верный ответ увеличивает прогресс и загадывает следующее слово.
- Ошибка **не сбрасывает прогресс**, но загадывает новое слово
  (и штрафует очками).
- Загаданные слова в рамках одной игры не повторяются. Если из-за ошибок
  словарь исчерпан, список использованных сбрасывается, но одно и то же
  слово никогда не выпадает два раза подряд.

## Сложность

| Уровень | Длина слов | Верных ответов до победы |
|---------|------------|--------------------------|
| 1       | 4–5 букв   | 3                        |
| 2       | 6–7 букв   | 4                        |
| 3       | 8+ букв    | 5                        |

Число ответов до победы: `roundCountForDifficulty(d) = d + 2`
(значения вне 1..3 приводятся к ближайшему уровню).

## Словари

Словари заданы прямо в коде (`logic/ReverseWords.kt`), по 34 слова на
уровень. Принципы:

- только существительные в именительном падеже единственного числа,
  повседневная лексика — спросонья слово должно узнаваться сразу;
- строчные буквы, «ё» не заменяется на «е»;
- длина слова строго соответствует уровню (4–5 / 6–7 / 8+);
- нет дубликатов и нет палиндромов (иначе перевёртыш совпал бы с ответом);
- слова подобраны «семьями» с общими буквами (коса/коза/кора,
  корова/ворона/сорока/корона, полка/палка/белка/булка), чтобы у каждого
  слова находились правдоподобные дистракторы.

## Подбор вариантов (дистракторов)

К правильному ответу добавляются 3 дистрактора из того же словаря
(`ReverseGame.pickDistractors`):

1. исключаются сам ответ и слово, совпадающее с его перевёртышем, —
   поэтому **ни один вариант никогда не равен слову на экране**
   (нет неоднозначности вида «кот» ↔ «ток»);
2. предпочитаются слова той же длины ±1;
3. кандидаты ранжируются по похожести:
   `общие_буквы × 2 + общее_начало × 3` (общие буквы считаются с учётом
   повторов — пересечение мультимножеств);
4. из 8 лучших кандидатов случайно берутся 3 — варианты остаются похожими
   на ответ, но не повторяются из раза в раз.

Все 4 варианта уникальны; порядок кнопок случайный.

## Очки

Как в остальных играх (образец — калькулятор):

- ошибка: **−10** очков;
- победа: **+ (600 − прошло_секунд) × difficulty**.

## Архитектура

```
feature/games/reverse/
├── src/main/java/com/example/smartalarm/feature/games/reverse/
│   ├── logic/ReverseWords.kt      # словари по уровням (чистый Kotlin)
│   ├── logic/ReverseGame.kt       # раунды, варианты, проверка, прогресс
│   │                              # (+ commonLetterCount, commonPrefixLength)
│   ├── ReverseGameViewModel.kt    # таймер «м.сс», очки, перезапуск будильника
│   └── ReverseGameFragment.kt     # экран: перевёртыш, 4 кнопки, навигация
├── src/main/res/layout/fragment_reverse_game.xml
├── src/test/java/.../logic/       # юнит-тесты чистой логики
└── README.md
```

- **Чистая логика** (`logic/`) не импортирует ничего из Android;
  случайность передаётся параметром `kotlin.random.Random` — тесты
  детерминированы через seed.
- **`ReverseGame`** — состояние игры: `currentRound` (ответ, перевёртыш,
  варианты), `correctCount`, `totalRounds`, `submitAnswer(index)` →
  `CORRECT` / `WRONG` / `GAME_WON`.
- **`ReverseGameViewModel`** — тонкий слой над `ReverseGame`
  (по образцу `CalcGameViewModel`): таймер раз в секунду, очки, загрузка
  будильника из Room и его перезапуск, если игра не пройдена. Игра живёт
  во ViewModel, поэтому пересоздание фрагмента не сбрасывает прогресс.

## Контракт экрана

Аргументы фрагмента (ключи с пробелами — исторические): `alarm id` (Long),
`start time` (Long, 0 = проба), `test` (Boolean), `difficulty` (Int 1..3),
`music path` (String).

- `onResume`: если не `test` — снимается уведомление будильника
  (id = `alarm id`).
- `onPause`: если не `test` и игра не пройдена — будильник перезапускается
  (`viewModel.startNewAlarm()`).
- На каждой чётной ненулевой минуте (секунды 00) и не в `test` — заново
  включается мелодия (`AlarmMediaPlayer.playAudio`) и показывается кнопка
  выключения музыки.
- Системная «назад»: в `test` — возврат к выбору игр
  (`action_reverseGameFragment2_to_gameChoiceFragment`); при будильнике —
  ничего не делает.
- Победа: в аргументы кладутся `time` (String), `score` (Int),
  `game id` (28), затем переход к результату:
  `action_reverseGameFragment2_to_gameResultFragment2` (test) или
  `action_reverseGameFragment_to_gameResultFragment` (будильник,
  плюс `viewModel.setPositiveResult()`).

Destination'ы объявлены в обоих nav-графах `:core:ui` (`nav_graph.xml`,
`nav_game_graph.xml`) и в демо-графе.

## Тесты

`src/test/java/com/example/smartalarm/feature/games/reverse/logic/`:

- **`ReverseWordsTest`** — длины слов по уровням, минимум 25 слов,
  отсутствие дубликатов и палиндромов, строчная кириллица,
  маппинг `forDifficulty`.
- **`ReverseGameTest`** — перевёртыш равен `answer.reversed()`; 4 уникальных
  варианта с ответом; ни один вариант не равен показанному слову; длины
  дистракторов ±1; детерминизм с seed; прогресс/ошибки/победа;
  неповторяемость слов за игру (включая ошибки); сброс исчерпанного словаря
  без повтора подряд; счётчики раундов 3/4/5; отказ от слишком малого
  словаря и неверного индекса.

Запуск: `./gradlew :feature:games:reverse:test`.

## Демо

Игра включена в демо-приложение со всеми мини-играми (без будильника
и Firebase):

```bash
./gradlew :feature:games:demo:installDebug
```

В лаунчере демо выберите «Наоборот» и уровень сложности.

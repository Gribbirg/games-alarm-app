# Мини-игра «Проценты»

Модуль `:feature:games:percent` (game id = **26**, пакет
`com.example.smartalarm.feature.games.percent`) — мини-игра будильника
SmartAlarm: чтобы выключить будильник, нужно решать задачи на проценты.

## Правила

- На экране показывается текстовая задача на проценты; ответ **всегда
  целое положительное число**.
- Под задачей четыре кнопки-варианта; ровно один вариант верный.
- Для победы нужно набрать N верных ответов. Ошибка **не сбрасывает**
  прогресс, но стоит **−10 очков**; после любого ответа (кроме
  победного) задаётся новая задача.
- Две подряд идущие задачи никогда не совпадают.

## Сложности и типы задач

| Сложность | Верных ответов | Типы задач |
|-----------|----------------|------------|
| 1 | 3 | «Сколько будет P% от X?» с «простыми» процентами P ∈ {10, 25, 50, 100} от круглых чисел (для 10% и 100% X кратно 10, для 25% — кратно 20, для 50% — чётное) |
| 2 | 4 | «Сколько будет P% от X?» с любым P, кратным 5 (5..95); «Число X увеличили/уменьшили на P%. Какое число получилось?» |
| 3 | 5 | Наценки и скидки: «Товар стоил X ₽, подорожал/подешевел на P%. Сколько он стоит теперь?»; обратные задачи: «P% числа равно Y. Найдите это число.» |

Диапазоны данных чисел: до 300 на сложности 1, до 400 (проценты) и
до 300 (изменение числа) на сложности 2; цены 50..1500 ₽ и искомые
числа до 400 на сложности 3. Проценты наценок/скидок берутся из
«магазинного» набора {10, 15, 20, 25, 30, 40, 50, 60, 70, 75, 80, 90}.

## Гарантия целого ответа

Условие строится **от целых частей**, а не проверяется перебором:
данное число X (в обратной задаче — искомое число) выбирается кратным
`100 / НОД(P, 100)`, поэтому `X·P` всегда делится на 100 нацело.
Например, для P = 15 НОД(15, 100) = 5, и X берётся кратным 20 —
15% от такого X целые. В обратной задаче известное значение
Y = X·P/100 вычисляется из уже выбранного целого ответа X.

## Дистракторы

Три неверных варианта — типичные ошибки решения плюс близкие значения:

- **P% от X**: дополнение `X − ответ` (взяли (100−P)%), само X,
  удвоенный/половинный ответ;
- **увеличение/наценка**: `X − прибавка` (изменили не в ту сторону),
  сама прибавка, X без изменений, `ответ + прибавка` (применили дважды);
- **уменьшение/скидка**: симметрично (`X + скидка`, скидка, X,
  `ответ − скидка`);
- **обратная задача**: `Y·P/100` (применили процент ещё раз), само Y,
  `2Y`, удвоенный ответ;
- для всех типов — значения на ±10 % и ±25 % от ответа.

Кандидаты фильтруются (только положительные, не равные ответу,
уникальные), из них случайно берутся три; при слишком бедном пуле
добираются случайными смещениями от ответа. Все четыре варианта
уникальны и положительны, порядок перемешивается.

## Очки

Как в игре «Калькулятор»:

- каждая ошибка: **−10**;
- при победе добавляется **(600 − прошло_секунд) × сложность**.

## Архитектура модуля

| Файл | Роль |
|------|------|
| `PercentLogic.kt` | Чистая логика партии: генерация задач с гарантированно целым ответом, дистракторы, прогресс, исходы ответов (`PercentGame`, `PercentTask`, `PercentTaskType`, `AnswerResult`). Случайность инжектируется через `kotlin.random.Random` (детерминизм в тестах). |
| `PercentGameViewModel.kt` | `AndroidViewModel`: таймер («м.сс»), очки, загрузка/перезапуск будильника (`AlarmDbRepository`, `AlarmCreateRepository`), тонкая обёртка над `PercentGame` с публикацией `LiveData`. |
| `PercentGameFragment.kt` | Экран игры: контракт экрана, привязка `LiveData` к UI, обработка ответов, навигация. |
| `res/layout/fragment_percent_game.xml` | Разметка: таймер, прогресс, карточка задачи, 4 кнопки-варианта, кнопка выключения музыки. |

Чистая логика (`PercentLogic.kt`) не импортирует `android`/`androidx`
и полностью покрывается JVM-тестами.

## Контракт экрана игры

Соблюдается общий контракт (`.claude/rules/games.md`):

- аргументы: `alarm id` (Long), `start time` (Long, 0 = проба),
  `test` (Boolean), `difficulty` (Int 1..3), `music path` (String);
- `onResume`: снятие уведомления будильника (кроме `test`);
- `onPause`: перезапуск будильника через `viewModel.startNewAlarm()`,
  если игра не пройдена (кроме `test`);
- каждые две минуты бездействия — повторное включение мелодии
  (`AlarmMediaPlayer.playAudio`) и показ кнопки «Выключить музыку»;
- системная «назад»: в режиме `test` — возврат к выбору игр
  (`action_percentGameFragment2_to_gameChoiceFragment`), при
  будильнике — ничего;
- победа: в аргументы кладутся `time` (String), `score` (Int),
  `game id` (Int = 26), затем переход к результату:
  `action_percentGameFragment2_to_gameResultFragment2` (проба) или
  `action_percentGameFragment_to_gameResultFragment` (будильник, плюс
  `viewModel.setPositiveResult()`).

## Тесты

`src/test/java/.../percent/PercentGameTest.kt` (JUnit 4, только чистая
логика):

- цель по сложностям (3/4/5 верных ответов);
- типы задач и проценты по сложностям (включая «круглость» чисел на
  сложности 1 и появление всех типов на 2 и 3);
- ответы целые, положительные и арифметически верные — пересчитываются
  в тесте из параметров задачи (тип, процент, данное число);
- текст задачи точно соответствует её параметрам;
- варианты: ровно 4, уникальны, положительны, содержат верный ответ;
- две подряд идущие задачи различны;
- детерминизм: одинаковый seed даёт одинаковую партию;
- прогресс: верные ответы ведут к победе, ошибка не сбрасывает
  прогресс, считается и меняет задачу, победа достижима после ошибки;
- неверный индекс варианта отвергается (`IllegalArgumentException`).

Запуск: `./gradlew :feature:games:percent:test`

без будильника и Firebase):

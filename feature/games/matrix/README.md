# :feature:games:matrix — игра «Запомни клетки»

Мини-игра будильника SmartAlarm (id игры в `ALL_GAMES` — **22**).
Пакет: `com.example.smartalarm.feature.games.matrix`.

## Описание и правила

На экране — квадратное поле из клеток (memory matrix). Игра идёт раундами:

1. Фаза показа (статус «Запоминай!»): весь набор клеток раунда **одновременно**
   вспыхивает на 1,5–2 секунды и гаснет; нажатия игрока в это время
   игнорируются.
2. Фаза воспроизведения (статус «Повтори!»): игрок отмечает по памяти те же
   клетки нажатиями. **Порядок не важен** — этим игра отличается от
   «Повтори узор» (`:feature:games:memory`), где клетки подсвечиваются по
   очереди и повторить их нужно строго в той же последовательности.
   Верно отмеченные клетки остаются подсвеченными; повторное нажатие на уже
   отмеченную клетку просто игнорируется (без штрафа).
3. Когда найдены все клетки набора — раунд пройден; в следующем раунде набор
   на одну клетку больше (до потолка сложности) и генерируется заново.
4. Ошибка (отмечена клетка, которая не вспыхивала) — штраф **−10 очков**,
   и текущий раунд начинается заново с **новым случайным набором** того же
   размера (показ повторяется).
5. Победа — собран набор последнего раунда. После победы игрок попадает на
   экран результата, будильник считается выключенным.

## Сложность

Уровень сложности (аргумент `difficulty`, 1..3; значения вне диапазона
приводятся к ближайшей границе) задаёт размер поля, число раундов, размер
набора и время показа (`MatrixGameSettings.forDifficulty`):

| Сложность | Поле | Раундов | Набор в 1-м раунде | Потолок набора | Показ, мс |
|-----------|------|---------|--------------------|----------------|-----------|
| 1         | 3×3  | 3       | 3                  | 5              | 1500      |
| 2         | 4×4  | 4       | 5                  | 8              | 1800      |
| 3         | 5×5  | 5       | 7                  | 10             | 2000      |

Размер набора в раунде `r`: `старт + (r − 1)`, но не больше потолка
(`MatrixGameSettings.cellsForRound`). Наборы по раундам: сложность 1 —
3, 4, 5 клеток; сложность 2 — 5, 6, 7, 8; сложность 3 — 7, 8, 9, 10, 10
(пятый раунд упирается в потолок). Потолок всегда меньше числа клеток поля,
поэтому «ошибочная» клетка существует в любом раунде.

## Очки

По образцу игры «Арифметика» (`:feature:games:calc`):

- каждая ошибка: **−10** очков (`MatrixGameLogic.MISTAKE_PENALTY`);
- при победе: **+ (600 − прошло_секунд) × сложность**.

Итог: `finalScore = −10 × ошибок + (600 − секунд_с_начала) × difficulty`
(`MatrixGameLogic.finalScore`). Счёт может быть отрицательным, если игра
длилась дольше 10 минут.

## Архитектура модуля

| Файл | Роль |
|------|------|
| `MatrixGameLogic.kt` | Чистая логика без Android-зависимостей: генерация набора уникальных клеток, приём отметок (`onCellTapped` → `MatrixTapResult`: верная / уже отмеченная / ошибка / раунд пройден / победа), перезапуск раунда с новым набором после ошибки, переход раундов (`startNextRound`), подсчёт ошибок и очков. Случайность — через параметр `kotlin.random.Random`, поэтому с seed поведение детерминировано (важно для тестов). |
| `MatrixGameSettings.kt` | Параметры сложности (таблица выше) и рост набора по раундам, чистый Kotlin. |
| `MatrixGameViewModel.kt` | Тонкий слой над логикой: хранит `MatrixGameLogic` и текущую фазу (показ/воспроизведение) — обе переживают поворот экрана (в фазе показа после поворота набор показывается заново на полное время, в фазе воспроизведения восстанавливаются отметки), секундный таймер «м.сс», загрузка/перезапуск будильника (`AlarmDbRepository`, `AlarmCreateRepository`), итоговый счёт. |
| `MatrixGameFragment.kt` | Экран игры: программно строит поле нужного размера из `MaterialCardView` в `android.widget.GridLayout` (весами на равномерное растяжение), подсветка набора и отметок цветами из атрибутов темы (`colorPrimaryContainer` — базовый, `colorPrimary` — показ, `colorTertiary` — отметки, `colorError` — ошибка), контракт экрана игры и навигация. Смена раунда/набора фиксируется в логике сразу, а пауза перед новым показом — только визуальная, поэтому поворот экрана в паузу не теряет прогресс. |
| `res/layout/fragment_matrix_game.xml` | Разметка: таймер, номер раунда, статус, квадратный `GridLayout` (клетки добавляются кодом), кнопка выключения музыки. |

## Контракт экрана (см. `.claude/rules/games.md`)

Аргументы фрагмента (ключи с пробелами — исторические):

- `alarm id` (Long), `start time` (Long, 0 = проба), `test` (Boolean),
  `difficulty` (Int 1..3), `music path` (String).

Поведение:

- `onResume`: если не `test` — снимается уведомление будильника
  (id уведомления = `alarm id`);
- `onPause`: если не `test` — будильник перезапускается, когда игра
  не пройдена (`viewModel.startNewAlarm()`);
- каждые две минуты бездействия (чётная ненулевая минута таймера, секунды 00)
  заново включается мелодия (`AlarmMediaPlayer.playAudio`) и появляется кнопка
  «Выключить музыку» (`AlarmMediaPlayer.stopAudio()` по нажатию);
- системная «назад»: в режиме пробы (`test == true`) — возврат к выбору игр
  (`action_matrixGameFragment2_to_gameChoiceFragment`); в режиме будильника —
  ничего не делает (в `nav_game_graph` экрана выбора игр нет);
- победа: в аргументы кладутся `time` (String, значение таймера),
  `score` (Int), `game id` (Int = 22), затем переход к результату:
  - проба: `action_matrixGameFragment2_to_gameResultFragment2` (`nav_graph`),
  - будильник: `action_matrixGameFragment_to_gameResultFragment`
    (`nav_game_graph`) плюс `viewModel.setPositiveResult()`.

Destination'ы объявлены в обоих nav-графах `:core:ui` и в графе демо;
модуль подключён в `settings.gradle.kts`, `app/build.gradle.kts` и
`feature/games/demo/build.gradle.kts`, ветки id = 22 есть в
`LoadGameFragment` и `GameChoiceFragment` хаба `:feature:games`.

## Тесты

JVM-тесты (JUnit 4, только чистая логика, без Android) в
`src/test/java/com/example/smartalarm/feature/games/matrix/`:

- `MatrixGameSettingsTest` — параметры всех трёх сложностей, приведение
  значений вне диапазона, рост набора по раундам с потолком и гарантия,
  что набор всегда меньше поля;
- `MatrixGameLogicTest` — размер и уникальность набора по сложностям и
  раундам, диапазон индексов, детерминизм с одинаковым seed, приём верных /
  повторных / ошибочных отметок, сброс отметок и новый набор после ошибки
  (тот же раунд, тот же размер), завершение раунда при сборе всего набора,
  победа на последнем раунде, запрет лишнего `startNextRound`, отметка вне
  поля, формула очков.

Запуск:

```bash
./gradlew :feature:games:matrix:test
```

## Запуск в демо-приложении

Игру можно запустить без будильника и Firebase из демо-приложения
(в основной APK оно не входит):

```bash
./gradlew :feature:games:demo:installDebug
```

В демо выбирается сложность и запускается «Запомни клетки» одной кнопкой
(режим пробы: уведомления, перезапуск будильника и музыка не задействованы).

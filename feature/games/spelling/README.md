# Игра «Как пишется?» (`:feature:games:spelling`)

Мини-игра для будильника SmartAlarm (game id = **30**): на экране показываются
2–4 варианта написания одного русского слова, из которых ровно **один
правильный**. Игрок выбирает вариант; чтобы выключить будильник, нужно набрать
заданное число верных ответов.

## Правила

- Каждый вопрос — варианты написания одного слова (правильное написание +
  1–3 неправильных, заранее составленных вручную). Порядок вариантов
  перемешивается случайно.
- Верный ответ увеличивает прогресс; после него показывается новое слово.
- Неверный ответ — штраф очков и тоже новое слово (то же слово повторно не
  показывается).
- Слова в рамках одной игры **не повторяются**. Если из-за большого числа
  ошибок пул уровня исчерпан, использованные записи сбрасываются, и слова
  начинают выдаваться по новому кругу (иначе игру было бы невозможно пройти).
- Победа — при наборе нужного числа верных ответов.

## Уровни сложности

Пул уровня N содержит **только** записи уровня N (уровни не смешиваются) —
так сложность вопросов чётко соответствует выбранному в будильнике уровню.
В каждом уровне не менее 20 записей.

| Уровень | Верных ответов для победы | Характер слов | Примеры записей (правильное / неправильные) |
|---|---|---|---|
| 1 | 4 | простые частотные слова школьного уровня | корова / карова; молоко / малоко, молако, малако; заяц / заец, заиц |
| 2 | 5 | средние орфограммы, «словарные» слова | винегрет / венигрет, венегрет, винигрет; искусство / искуство, исскуство; аппетит / апетит, аппитит |
| 3 | 6 | сложные случаи, в которых ошибаются и взрослые | прийти / придти, притти; в течение недели / в течении недели; серебряный / серебрянный; деревянный / деревяный |

Записи с одинаково пишущимися, но разными по смыслу формами снабжены
поясняющим контекстом прямо в строке варианта — например, «ветреный (день)»,
«мороженое (десерт)», «в течение недели» — чтобы правильный ответ был
однозначен.

## Очки (по образцу игры-калькулятора)

- каждая ошибка: **−10** очков (`SpellingGame.MISTAKE_PENALTY`);
- при победе: **`+ (600 − прошло_секунд) × difficulty`**
  (`SpellingGame.finishBonus`) — чем быстрее и чем сложнее уровень, тем больше
  очков; при игре дольше 10 минут бонус становится отрицательным.

## Архитектура

Чистая логика отделена от Android-кода и не содержит import'ов
`android`/`androidx`; вся случайность передаётся параметром
`kotlin.random.Random`, поэтому логика детерминированно тестируется с seed.

| Файл | Роль |
|---|---|
| `SpellingEntry.kt` | Запись базы: `correct` + список `wrong` (1–3 варианта). Чистый Kotlin. |
| `SpellingQuestion.kt` | Готовый вопрос: перемешанные `options` + `correctIndex`. Чистый Kotlin. |
| `AnswerResult.kt` | Результат ответа: `CORRECT` / `WRONG` / `WIN`. Чистый Kotlin. |
| `SpellingData.kt` | База слов: `LEVEL_1`/`LEVEL_2`/`LEVEL_3` (≥20 записей каждый) и `entriesForDifficulty()`. Чистый Kotlin. |
| `SpellingGame.kt` | Движок игры: выбор неповторяющейся записи, перемешивание вариантов, проверка ответа, прогресс/победа, константы очков. Чистый Kotlin. |
| `SpellingGameViewModel.kt` | `AndroidViewModel`: таймер («м.сс»), очки, загрузка/перезапуск будильника (`AlarmDbRepository`, `AlarmCreateRepository`) — тонкий слой над `SpellingGame`; вопрос и прогресс отдаёт через `LiveData`. |
| `SpellingGameFragment.kt` | Экран игры: рендер вариантов (2–4 кнопки), обработка ответов, контракт экрана игры. |
| `res/layout/fragment_spelling_game.xml` | Разметка: таймер, счётчик прогресса, заголовок, карточка с 4 кнопками-вариантами (лишние скрываются), кнопка выключения музыки. Цвета — через `?attr/...`. |

## Контракт экрана игры

Реализован по образцу `CalcGameFragment` (см. `.claude/rules/games.md`):

- аргументы: `alarm id` (Long), `start time` (Long, 0 = проба), `test`
  (Boolean), `difficulty` (Int 1..3), `music path` (String);
- `onResume`: если не `test` — снимается уведомление будильника;
- `onPause`: если не `test` и игра не пройдена — будильник перезапускается;
- каждые две минуты бездействия заново включается мелодия
  (`AlarmMediaPlayer.playAudio`) и появляется кнопка её выключения;
- системная «назад»: в режиме `test` — возврат к выбору игр
  (`action_spellingGameFragment2_to_gameChoiceFragment`), при реальном
  будильнике — игнорируется (будильник нельзя «сбежать» кнопкой назад);
- победа: в аргументы кладутся `time` (String), `score` (Int),
  `game id` (Int = 30) и выполняется переход к результату
  (`..._to_gameResultFragment2` в `test`, `..._to_gameResultFragment` —
  при будильнике, с `setPositiveResult()`).

## Тесты

`src/test/java/...` (JUnit4, только чистая логика):

- `SpellingDataTest` — на каждом уровне ≥20 записей; в каждой записи 1–3
  wrong-варианта, они не равны эталону, уникальны и не пусты; эталоны внутри
  уровня не повторяются; маппинг `entriesForDifficulty` c приведением
  значений вне 1..3;
- `SpellingGameTest` — в вопросе ровно один правильный вариант и все варианты
  взяты из одной записи; перемешивание детерминировано с одинаковым seed;
  записи не повторяются за игру; прогресс и `WIN`; ошибки (`WRONG`, счётчик,
  новое слово); переиспользование пула при исчерпании; неизменность состояния
  после победы; формулы `targetCorrectFor` и `finishBonus`; проходимость всех
  трёх реальных уровней базы.

Запуск: `./gradlew :feature:games:spelling:test`.

## Запуск в демо

Игра включена в демо-приложение со всеми мини-играми (без будильника и
Firebase):

```bash
./gradlew :feature:games:demo:installDebug
```

В списке демо игра называется «Как пишется?»; уровень сложности выбирается
на экране запуска.

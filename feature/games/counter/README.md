# Игра «Сосчитай» (`:feature:games:counter`)

Мини-игра будильника SmartAlarm, **game id = 13** (см. `ALL_GAMES` в `:core:data`).

## Правила

На экране — квадратная сетка, заполненная вперемешку эмодзи нескольких
видов, и вопрос «Сколько на поле 🍎?». Нужно сосчитать целевые эмодзи и
выбрать один из четырёх вариантов ответа. Верный ответ открывает следующий
раунд; неверный — штрафуется, а поле **генерируется заново с другим целевым
эмодзи** (номер раунда не меняется), поэтому перебор четырёх вариантов не
работает. После последнего раунда игра пройдена, будильник можно выключить.

## Сложности

| Сложность | Сетка | Раундов | Видов эмодзи | Похожесть видов |
|-----------|-------|---------|--------------|-----------------|
| 1 | 4×4 | 3 | 3 | виды из трёх разных групп — заметно отличаются |
| 2 | 5×5 | 4 | 4 | по два вида из двух групп — частично похожи |
| 3 | 6×6 | 5 | 5 | все виды из одной группы — визуально похожи |

Сторона сетки — `difficulty + 3`, число раундов и видов — `difficulty + 2`.
Значение сложности вне 1..3 приводится к границам диапазона.

## Генерация поля

- Число целевых эмодзи выбирается равновероятно из диапазона
  `3 .. (ячеек / 2 − 1)`: всегда **не меньше 3** и **строго меньше половины
  поля** (4×4 → 3..7, 5×5 → 3..11, 6×6 → 3..17).
- Каждый вид-дистрактор появляется на поле **хотя бы один раз**, остальные
  ячейки заполняются дистракторами случайно; затем всё поле перемешивается.
- Целевой эмодзи не повторяется два раунда подряд (в том числе после
  перегенерации из-за ошибки).

## Варианты ответа

Четыре уникальных числа в случайном порядке: правильный ответ плюс три из
шести соседей `правильный ± 1..3`. Так как правильный ответ всегда ≥ 3, все
варианты неотрицательны.

## Пул эмодзи

Пул объявлен в `CounterEmoji.kt` (`COUNTER_EMOJI_GROUPS`): 8 групп по 5
визуально похожих символов, все эмодзи глобально уникальны.

| Группа | Эмодзи |
|--------|--------|
| красное и круглое | 🍎 🍅 🔴 🎈 🍓 |
| морды зверей | 🐶 🐺 🦊 🐱 🐯 |
| мячи | ⚽ 🏐 🏀 ⚾ 🎱 |
| улыбки | 😀 😃 😄 😁 🙂 |
| цветы | 🌸 💮 🌼 🌻 🌺 |
| деревья | 🌲 🌳 🌴 🎄 🌵 |
| сердца | ❤️ 🧡 💛 💚 💙 |
| луны | 🌕 🌝 🌛 🌜 🌙 |

На сложности 1 берётся по одному виду из трёх случайных групп, на 2 — по два
вида из двух групп, на 3 — все пять видов одной группы (считать похожие
эмодзи на поле 6×6 заметно труднее). Чтобы добавить группу, достаточно
дописать `EmojiGroup(имя, 5+ эмодзи)` — логика и тесты подстроятся
автоматически.

## Очки

Формула по образцу арифметики (`:feature:games:calc`):

```
итог = −10 × ошибки + (600 − прошло_секунд) × сложность
```

- каждый неверный ответ: −10 очков;
- на финише: + (600 − длительность игры в секундах) × difficulty.

Чистая функция — `computeCounterScore()` в `CounterGame.kt`.

## Архитектура

| Файл | Роль |
|------|------|
| `CounterEmoji.kt` | Чистая логика: модель `EmojiGroup` и пул `COUNTER_EMOJI_GROUPS`. Без зависимостей от Android. |
| `CounterGame.kt` | Чистая логика: `CounterRound` (поле, целевой эмодзи, правильный ответ, варианты), `CounterGame` (выбор видов по сложности, генерация поля и вариантов, раунды, ошибки, победа), `AnswerResult`, `computeCounterScore()`. Случайность инжектируется через `kotlin.random.Random`. |
| `CounterGameViewModel.kt` | `AndroidViewModel`: таймер игры (строка «м.сс»), загрузка будильника (`AlarmDbRepository`/`AlarmsDB`) и его перезапуск (`AlarmCreateRepository`), тонкие делегаты к `CounterGame`, итоговый счёт. |
| `CounterGameFragment.kt` | UI: строит сетку эмодзи программно в `android.widget.GridLayout` (ячейки — `TextView` с весами), показывает вопрос и четыре кнопки-варианта, контракт экрана игры. |
| `res/layout/fragment_counter_game.xml` | Разметка: таймер, вопрос, счётчик раундов, карточка с `GridLayout`, ряд из четырёх кнопок, кнопка выключения музыки. Цвета — через `?attr/...` темы. |

## Контракт экрана (см. `.claude/rules/games.md`)

Аргументы фрагмента: `alarm id` (Long), `start time` (Long, 0 = проба),
`test` (Boolean), `difficulty` (Int 1..3), `music path` (String).

- `onResume`: если не `test` — снимается уведомление будильника;
- `onPause`: если не `test` и игра не пройдена — будильник перезапускается;
- каждые две минуты бездействия заново включается мелодия
  (`AlarmMediaPlayer`) и появляется кнопка её выключения;
- системная «назад»: в режиме `test` — возврат к выбору игр
  (`action_counterGameFragment2_to_gameChoiceFragment`), при реальном
  будильнике — игнорируется;
- при победе в аргументы кладутся `time` (String), `score` (Int),
  `game id` = 13 и происходит переход к результату
  (`action_counterGameFragment2_to_gameResultFragment2` в пробе,
  `action_counterGameFragment_to_gameResultFragment` при будильнике).

Destination'ы объявлены в обоих nav-графах `:core:ui` и в демо-графе;
ветки для id 13 есть в `LoadGameFragment` и `GameChoiceFragment`
(`:feature:games`).

## Тесты

`src/test/java/.../CounterGameTest.kt` (JUnit4, только чистая логика):

- пул: ≥3 групп, в каждой ≥5 уникальных эмодзи, глобальная уникальность;
- сложности: сетки 4×4/5×5/6×6, раунды 3/4/5, виды 3/4/5, приведение
  сложности к 1..3;
- поле: заявленный правильный ответ равен фактическому числу целевых,
  размер поля `сторона²`, целевых ≥3 и строго меньше половины, ровно
  `kindCount` видов и каждый дистрактор хотя бы один раз, все эмодзи из
  пула, число групп-источников по сложностям (3/2/1);
- варианты: 4 уникальных, содержат правильный, все ≥ 0;
- детерминизм: одинаковый seed даёт одинаковую игру;
- ответы: верный — следующий раунд, неверный — счётчик ошибок без смены
  номера раунда и перегенерация поля с другим целевым эмодзи, целевой не
  повторяется два раунда подряд, победа после всех раундов, ответы после
  победы игнорируются, ошибки накапливаются;
- счёт: формула, −10 за ошибку, возможность отрицательного счёта.

Запуск: `./gradlew :feature:games:counter:test`.

## Запуск в демо

Игра включена в демо-приложение со всеми мини-играми (без будильника и
Firebase):

```bash
./gradlew :feature:games:demo:installDebug
```

В лаунчере демо выберите «Сосчитай» и сложность.

# Игра «Ханойская башня» (`:feature:games:hanoi`)

Мини-игра будильника SmartAlarm, id игры в `ALL_GAMES` — **20**.

## Правила

Три стержня; на первом (левом) собрана башня из N дисков разного размера —
внизу самый большой. Нужно перенести всю башню на третий (правый) стержень:

- за один ход перекладывается один верхний диск;
- класть больший диск на меньший **нельзя**;
- победа — все N дисков собраны на третьем стержне (сборка башни на
  среднем стержне победой не считается).

Минимально возможное число ходов — классические `2^N − 1`; оно показано
рядом со счётчиком: «Ходы: k (мин. M)».

## Управление

- Тап по стержню-источнику — его верхний диск выделяется (поднимается над
  стержнем и подсвечивается акцентным цветом).
- Тап по стержню-цели — ход выполняется.
- Повторный тап по стержню-источнику — выделение снимается.
- Тап по пустому стержню, когда ничего не выбрано, игнорируется (это не
  попытка хода, штрафа нет).
- Недопустимый ход (больший диск на меньший) — **−10 очков** и Toast
  «Нельзя класть больший диск на меньший!»; выделение при этом
  сохраняется, чтобы можно было сразу выбрать другую цель.

## Сложность

| Уровень | Дисков | Минимум ходов |
|---------|--------|---------------|
| 1       | 3      | 7             |
| 2       | 4      | 15            |
| 3       | 5      | 31            |

## Очки

Как в остальных играх (образец — `:feature:games:calc`):

- каждый недопустимый ход: `−10`;
- при победе: `+ (600 − прошло_секунд) × сложность`.

Итог кладётся в аргумент `score` экрана результата.

## Архитектура (файлы и роли)

- `HanoiGameLogic.kt` — **чистая логика без Android-зависимостей**:
  три стержня (списки размеров дисков снизу вверх), `move(from, to)` →
  `MoveResult.OK / ILLEGAL / WIN`, счётчик ходов `moveCount` (нелегальные
  попытки не считаются и не меняют состояние), `minMoves = 2^N − 1`,
  `isWon`, снимок состояния `rodsSnapshot()` для UI и
  `disksForDifficulty()` (1→3, 2→4, 3→5 дисков). Старт детерминирован,
  случайности в игре нет.
- `HanoiView.kt` — кастомная View: рисует основание, три стержня и диски
  скруглёнными прямоугольниками; верхний диск выбранного стержня рисуется
  «поднятым» и акцентным цветом. Тап по любой из трёх вертикальных третей
  View отдаётся наружу через колбэк `onRodTapped(rod)` — View не знает
  правил игры. Цвета — из атрибутов темы (`colorPrimary`,
  `colorTertiary`, `colorOnBackground`) с контрастными fallback'ами.
- `HanoiGameViewModel.kt` — `AndroidViewModel` по образцу
  `CalcGameViewModel`: таймер «м.сс» (`timeCurrentString`), счёт и штрафы,
  загрузка будильника (`AlarmDbRepository`/`AlarmsDB`) и его перезапуск
  (`AlarmCreateRepository.create`), если игра не пройдена. Держит партию
  (`ensureGame()` создаёт её один раз) и выбранный стержень
  (`selectedRod`) — состояние переживает поворот экрана.
- `HanoiGameFragment.kt` — экран игры: связывает аргументы, ViewModel,
  `HanoiView` и навигацию; реализует контракт экрана игры (ниже).
- `res/layout/fragment_hanoi_game.xml` — таймер, инструкция, счётчик
  ходов, `HanoiView` и кнопка выключения музыки
  (`@drawable/outline_music_off_24` из `:core:ui`).

## Контракт экрана (см. `.claude/rules/games.md`)

Аргументы фрагмента: `alarm id` (Long), `start time` (Long, 0 — проба),
`test` (Boolean), `difficulty` (Int 1..3), `music path` (String).

- `onResume`: не в `test` — снимает уведомление будильника
  (`NotificationManagerCompat`, id = `alarm id`).
- `onPause`: не в `test` — `viewModel.startNewAlarm()` перезапускает
  будильник, если игра не пройдена.
- Каждая чётная ненулевая минута (секунды `00`), не в `test` — повторно
  включается мелодия (`AlarmMediaPlayer.playAudio`) и появляется кнопка
  «Выключить музыку» (`AlarmMediaPlayer.stopAudio()`).
- Системная «назад»: в `test` — возврат к выбору игр
  (`action_hanoiGameFragment2_to_gameChoiceFragment`,
  `popUpTo(alarmsFragment, inclusive)`); не в `test` (будильник) —
  ничего не происходит, игру нужно пройти.
- Победа: в аргументы кладутся `time` (String), `score` (Int),
  `game id` (Int = 20); в `test` — переход по
  `action_hanoiGameFragment2_to_gameResultFragment2`, иначе —
  `action_hanoiGameFragment_to_gameResultFragment` +
  `viewModel.setPositiveResult()`.

Destination'ы объявлены в обоих nav-графах `:core:ui`
(`nav_graph.xml` — проба, `nav_game_graph.xml` — будильник) и в демо-графе.

## Тесты

`src/test/java/.../HanoiGameLogicTest.kt` (JUnit4, только чистая логика):

- стартовое состояние (все диски на первом стержне по убыванию) и
  `minMoves` для N = 3, 4, 5;
- легальные ходы (перенос верхнего диска, меньший на больший);
- нелегальные ходы: больший на меньший, с пустого стержня, `from == to`,
  индексы вне 0..2 — состояние не меняется, **ход не засчитывается**;
- победа только на третьем стержне (башня на среднем — не победа);
- оптимальное решение рекурсивным алгоритмом приводит к победе ровно за
  `2^N − 1` ходов для N = 3, 4, 5 (последний ход возвращает `WIN`);
- соответствие сложности числу дисков.

Запуск: `./gradlew :feature:games:hanoi:test`.

## Запуск в демо

без будильника и Firebase):

# Мини-игра «Поймай момент»

Модуль `:feature:games:reaction`, game id = **12** (запись в `ALL_GAMES`),
пакет `com.example.smartalarm.feature.games.reaction`.

## Правила

По горизонтальной шкале туда-обратно равномерно бегает бегунок
(треугольная волна: от левого края к правому и обратно, без пауз).
На шкале подсвечена **целевая зона** — случайный отрезок. Задача игрока —
нажать большую кнопку **«Стоп»** в тот момент, когда бегунок находится
внутри зоны.

- Попадание засчитывается, если позиция бегунка в момент нажатия лежит
  в зоне, **границы включительно**.
- После каждого попадания зона **перегенерируется** в случайном месте
  шкалы (той же ширины), поэтому «запомнить ритм» одной точки нельзя.
- Промах отнимает очки, но зону не меняет — можно прицелиться заново.
- Победа — набрать нужное число попаданий (см. сложности).

Игра тренирует реакцию и концентрацию: чем выше сложность, тем быстрее
бегунок и уже зона.

## Сложности

| Сложность | Попаданий для победы | Ширина зоны | Период волны (туда-обратно) | Проход края-в-край |
|---|---|---|---|---|
| 1 | 3 | 20 % шкалы | 2400 мс | 1,2 с |
| 2 | 4 | 14 % шкалы | 1800 мс | 0,9 с |
| 3 | 5 | 10 % шкалы | 1300 мс | 0,65 с |

Значение сложности вне 1..3 приводится к ближайшей границе диапазона.
Окно попадания по времени ≈ `ширина_зоны × полупериод`: от ~240 мс на
сложности 1 до ~65 мс на сложности 3.

## Очки

- попадание: **+10**;
- промах: **−10**;
- при победе добавляется бонус за скорость по общей формуле игр:
  `(600 − прошло_секунд) × сложность`.

Итог: `10 × попадания − 10 × промахи + (600 − секунды) × сложность`.

## Архитектура

Чистая логика (без единого `import android*`, случайность — через параметр
`kotlin.random.Random`, детерминируется seed'ом в тестах):

- `ReactionWave.kt` — треугольная волна `position(elapsedMs, periodMs) → 0..1`:
  позиция бегунка — **чистая функция от прошедших миллисекунд**
  (`0 → 0`, `period/2 → 1`, `period → 0`), UI лишь спрашивает её по таймеру;
- `ReactionZone.kt` — целевая зона `[start, end]` внутри 0..1: генерация
  случайной зоны заданной ширины (`generate`) и проверка попадания
  (`contains`, границы включительно);
- `ReactionSettings.kt` — параметры уровней сложности (`forLevel`);
- `ReactionGame.kt` — состояние партии: приём нажатий «Стоп» (`press`),
  счётчики попаданий/промахов, очки, перегенерация зоны после попадания,
  признак победы; нажатия после победы игнорируются.

UI-слой:

- `ReactionScaleView.kt` — кастомная View с `onDraw`: дорожка, подсвеченная
  зона и вертикальный бегунок; цвета берутся из атрибутов темы
  (`colorSurfaceVariant`, `colorTertiary`, `colorOnBackground`) с
  контрастными fallback-значениями, поэтому шкала читаема в светлой и
  тёмной темах; View только рисует то, что ей задали (`setZone`,
  `setIndicatorPosition`);
- `ReactionGameViewModel.kt` — тонкая обёртка над `ReactionGame` по образцу
  калькулятора: таймер «м.сс», точка отсчёта волны `waveStartMs`
  (хранится во ViewModel — волна не сбивается при пересоздании фрагмента),
  загрузка/пересоздание будильника (`AlarmDbRepository`/
  `AlarmCreateRepository`), финальный счёт;
- `ReactionGameFragment.kt` + `res/layout/fragment_reaction_game.xml` —
  экран: таймер, задание с подсказкой, шкала, счётчик «Попаданий: k / N»,
  большая кнопка «Стоп», кнопка выключения музыки. Фрагмент тикает
  `Handler`'ом каждые ~16 мс (старт в `onResume`, остановка в `onPause`)
  и на каждый тик перерисовывает бегунок по `ReactionWave.position`.

## Контракт экрана игры

Реализован по `.claude/rules/games.md` (образец — `CalcGameFragment`):

- аргументы: `alarm id` (Long), `start time` (Long, 0 = проба),
  `test` (Boolean), `difficulty` (Int 1..3), `music path` (String);
- `onResume`: вне `test` снимается уведомление будильника;
- `onPause`: вне `test` будильник пересоздаётся, если игра не пройдена;
- каждые две минуты бездействия заново включается мелодия
  (`AlarmMediaPlayer`) и появляется кнопка её выключения;
- системная «назад»: в `test` — возврат к выбору игр
  (`action_reactionGameFragment2_to_gameChoiceFragment`), при реальном
  будильнике — игнорируется;
- при победе в аргументы кладутся `time` (String), `score` (Int),
  `game id` = 12 и выполняется переход к результату: в `test` —
  `action_reactionGameFragment2_to_gameResultFragment2`, иначе —
  `action_reactionGameFragment_to_gameResultFragment` +
  `viewModel.setPositiveResult()`.

Destination'ы объявлены в обоих nav-графах `:core:ui`
(`nav_graph.xml` — проба, `nav_game_graph.xml` — будильник) и в демо-графе.

## Тесты

`src/test/java/.../ReactionWaveTest.kt`, `ReactionZoneTest.kt`,
`ReactionSettingsTest.kt`, `ReactionGameTest.kt` (JUnit4, только чистая
логика):

- треугольная волна: опорные точки (`0 → 0`, `period/2 → 1`, `period → 0`),
  строгая монотонность на полупериодах, периодичность, диапазон 0..1,
  отрицательное время, невалидный период;
- зона: генерация в границах шкалы и точно заданной ширины для всех
  сложностей, включённость границ (`start`/`end` — попадание, чуть вне —
  промах), невалидные зоны/ширины отклоняются;
- параметры сложностей и приведение значений вне 1..3;
- партия: попадание/промах (очки, счётчики, перегенерация зоны только
  после попадания), попадания на границах зоны, победа ровно после
  `hitsToWin` попаданий, промахи не приближают победу, нажатия после
  победы игнорируются;
- детерминизм: одинаковый seed — одинаковые зоны и партии.

Запуск: `./gradlew :feature:games:reaction:test`.

## Запуск в демо

Игра включена в демо-приложение со всеми мини-играми (без будильника
и Firebase):

```bash
./gradlew :feature:games:demo:installDebug
```

На экране демо выберите «Поймай момент» и сложность.

# Пятнашки (`:feature:games:fifteen`)

Мини-игра «Пятнашки» (game id = **15**) для будильника SmartAlarm: чтобы выключить
будильник, нужно собрать классическую головоломку с плитками.

## Правила

- Поле `N × N` заполнено плитками с числами `1..N²−1`, одна клетка пустая.
- Тап по плитке, соседней с пустой клеткой (по горизонтали или вертикали),
  сдвигает её на пустое место. Тапы по остальным плиткам игнорируются.
- Цель — расставить плитки по порядку `1, 2, …, N²−1` слева направо и сверху
  вниз, пустая клетка — в правом нижнем углу.
- Ошибок в игре нет: неудачные тапы ничего не портят и не штрафуются,
  на счёт влияют только время и число ходов.
- На экране отображаются секундомер и счётчик выполненных ходов.

## Сложности

| Уровень | Поле | Ходов перемешивания K |
|--------:|:----:|:---------------------:|
| 1 | 3×3 | 15 |
| 2 | 3×3 | 40 |
| 3 | 4×4 | 60 |

Значения сложности вне диапазона 1..3 приводятся к ближайшей границе
(`FifteenDifficulty.forLevel`).

## Гарантия решаемости

Поле перемешивается **K случайными корректными ходами из собранного состояния**
(`FifteenBoard.shuffle`), а не случайной перестановкой плиток, поэтому:

- головоломка всегда решаема — обратная последовательность ходов возвращает
  поле в собранное состояние, то есть решение существует и требует не более K ходов;
- ход, немедленно отменяющий предыдущий, при перемешивании никогда не
  выбирается — K ходов не «схлопываются» в более короткую прогулку туда-обратно;
- если после K ходов поле случайно оказалось собранным, перемешивание
  выполняется заново — игрок никогда не получает уже решённое поле.

## Формула очков

Подсчёт — в `FifteenGameViewModel.finishScore()`:

```
очки = (600 − секунды с начала игры) × сложность + бонус за ходы
бонус за ходы = max(0, 2K − число ходов игрока)
```

Первое слагаемое — стандартная формула по времени (как в игре «Калькулятор»).
Бонус поощряет решение, близкое к минимальному числу ходов: перемешивание
K ходами гарантирует, что решение существует не длиннее K ходов, поэтому
решение ровно за K ходов даёт K бонусных очков, а за 2K ходов и более — ноль.

## Архитектура

Чистая логика (без `android.*`/`androidx.*`, покрыта юнит-тестами):

| Файл | Роль |
|------|------|
| `FifteenBoard.kt` | Поле: список плиток построчно (`0` — пустая клетка), позиция пустой клетки, проверка и выполнение ходов (`canMove`/`movableIndices`/`moveTile`), проверка сборки (`isSolved`), перемешивание K ходами (`shuffle`, случайность — через параметр `kotlin.random.Random`). `FifteenMove` — запись одного хода (откуда/куда), обратный ход — `moveTile(toIndex)`. |
| `FifteenDifficulty.kt` | Таблица сложностей: уровень → размер поля и K. |
| `FifteenGame.kt` | Партия: перемешанное поле, счётчик ходов игрока, признак победы `isWon`, бонус `moveBonus()`. |

Android-слой:

| Файл | Роль |
|------|------|
| `FifteenGameViewModel.kt` | `AndroidViewModel`: секундомер (строка «м.сс», LiveData), доступ к БД будильников (`AlarmDbRepository`), перезапуск будильника при уходе с экрана (`AlarmCreateRepository`), подсчёт очков. Экземпляр `FifteenGame` живёт здесь (`ensureGame`) и переживает поворот экрана. |
| `FifteenGameFragment.kt` | Экран игры: строит поле программно в `android.widget.GridLayout` (кнопки-плитки с весами, пустая клетка — невидимая кнопка), обрабатывает тапы, реализует контракт экрана игры. |
| `res/layout/fragment_fifteen_game.xml` | Разметка: секундомер, счётчик ходов, квадратный `GridLayout`, кнопка выключения музыки. Цвета — через атрибуты темы (`?attr/...`). |

## Контракт экрана

Аргументы фрагмента: `alarm id` (Long), `start time` (Long, 0 — проба),
`test` (Boolean), `difficulty` (Int 1..3), `music path` (String).

- `onResume`: если не `test` — снимается уведомление будильника
  (`NotificationManagerCompat`, id = `alarm id`).
- `onPause`: если не `test` и игра не пройдена — будильник перезапускается
  (`viewModel.startNewAlarm()`).
- Каждые две минуты бездействия мелодия включается заново
  (`AlarmMediaPlayer.playAudio`), появляется кнопка «Выключить музыку»
  (`AlarmMediaPlayer.stopAudio()`).
- Системная «назад»: в режиме `test` — возврат к выбору игр
  (`action_fifteenGameFragment2_to_gameChoiceFragment`), в режиме будильника —
  игнорируется (будильник нельзя обойти).
- Победа: в аргументы кладутся `time` (String), `score` (Int),
  `game id` (Int = 15); переход к результату — в `test` через
  `action_fifteenGameFragment2_to_gameResultFragment2`, иначе через
  `action_fifteenGameFragment_to_gameResultFragment` + `setPositiveResult()`.

## Тесты

`src/test/java/.../FifteenBoardTest.kt`, `FifteenGameTest.kt` (JUnit4, только
чистая логика): сдвигаются только соседи пустой клетки; обратный ход
восстанавливает состояние; перемешанное поле не собрано и решается обратной
последовательностью ходов; перемешивание не отменяет предыдущий ход;
детерминизм при одинаковом seed; размеры поля и K по уровням сложности;
победа при сборке; счётчик ходов и границы бонуса.

Запуск: `./gradlew :feature:games:fifteen:test`.

## Запуск в демо

Игра включена в демо-приложение со всеми мини-играми (без будильника и Firebase):

```bash
./gradlew :feature:games:demo:installDebug
```

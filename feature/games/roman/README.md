# Мини-игра «Римские числа»

Модуль `:feature:games:roman` (game id = **17**, пакет
`com.example.smartalarm.feature.games.roman`) — мини-игра будильника
SmartAlarm: чтобы выключить будильник, нужно переводить числа между
римской и арабской записью.

## Правила

- На экране показывается число в римской **или** арабской записи —
  направление каждого вопроса выбирается случайно.
- Под вопросом четыре кнопки-варианта с записями в «противоположной»
  системе; ровно один вариант верный.
- Для победы нужно набрать N верных ответов. Ошибка **не сбрасывает**
  прогресс, но стоит **−10 очков**; после любого ответа (кроме
  победного) задаётся новый вопрос.
- Два подряд идущих вопроса никогда не загадывают одно и то же число.

## Сложности

| Сложность | Диапазон чисел | Верных ответов для победы |
|-----------|----------------|---------------------------|
| 1         | 1..20          | 4                         |
| 2         | 1..100         | 5                         |
| 3         | 1..1000        | 6                         |

## Дистракторы

Три неверных варианта подбираются правдоподобными — из чисел,
отличающихся от загаданного:

- на **±1..10** (близкие значения);
- на **±40, ±50, ±90, ±100, ±400, ±500, ±900** — «римское расстояние»
  в один символ, такие числа дают визуально похожие римские записи
  (например, для 94 = XCIV кандидаты 44 = XLIV и 194 = CXCIV).

Кандидаты ограничиваются диапазоном сложности, из них случайно берутся
три различных; если пул слишком мал (на практике не бывает), добирается
случайными числами диапазона. Все четыре варианта уникальны, порядок
перемешивается.

## Конвертер римских чисел

`RomanNumerals` — чистый (без Android) конвертер:

- `toRoman(number)` — каноническая римская запись для 1..3999, включая
  вычитательную нотацию (IV, IX, XL, XC, CD, CM): 1994 → `MCMXCIV`.
- `fromRoman(roman)` — разбор по вычитательному правилу (меньший символ
  перед большим вычитается) с последующей проверкой каноничности
  обратным преобразованием: `IIII`, `VV`, `IL`, пустая строка и
  посторонние символы отвергаются (`IllegalArgumentException`).

## Очки

Как в игре «Калькулятор»:

- каждая ошибка: **−10**;
- при победе добавляется **(600 − прошло_секунд) × сложность**.

## Архитектура модуля

| Файл | Роль |
|------|------|
| `RomanNumerals.kt` | Чистый конвертер римская ↔ арабская запись. |
| `RomanGame.kt` | Чистая логика партии: генерация вопросов и дистракторов, прогресс, исходы ответов (`RomanGame`, `RomanQuestion`, `QuestionDirection`, `AnswerResult`). Случайность инжектируется через `kotlin.random.Random` (детерминизм в тестах). |
| `RomanGameViewModel.kt` | `AndroidViewModel`: таймер («м.сс»), очки, загрузка/перезапуск будильника (`AlarmDbRepository`, `AlarmCreateRepository`), тонкая обёртка над `RomanGame` с публикацией `LiveData`. |
| `RomanGameFragment.kt` | Экран игры: контракт экрана, привязка `LiveData` к UI, обработка ответов, навигация. |
| `res/layout/fragment_roman_game.xml` | Разметка: таймер, прогресс, карточка вопроса, 4 кнопки-варианта, кнопка выключения музыки. |

Чистая логика (`RomanNumerals.kt`, `RomanGame.kt`) не импортирует
`android`/`androidx` и полностью покрывается JVM-тестами.

## Контракт экрана игры

Соблюдается общий контракт (`.claude/rules/games.md`):

- аргументы: `alarm id` (Long), `start time` (Long, 0 = проба),
  `test` (Boolean), `difficulty` (Int 1..3), `music path` (String);
- `onResume`: снятие уведомления будильника (кроме `test`);
- `onPause`: перезапуск будильника через `viewModel.startNewAlarm()`,
  если игра не пройдена (кроме `test`);
- каждые две минуты бездействия — повторное включение мелодии
  (`AlarmMediaPlayer.playAudio`) и показ кнопки «Выключить музыку»;
- системная «назад»: в режиме `test` — возврат к выбору игр
  (`action_romanGameFragment2_to_gameChoiceFragment`), при будильнике —
  ничего;
- победа: в аргументы кладутся `time` (String), `score` (Int),
  `game id` (Int = 17), затем переход к результату:
  `action_romanGameFragment2_to_gameResultFragment2` (проба) или
  `action_romanGameFragment_to_gameResultFragment` (будильник, плюс
  `viewModel.setPositiveResult()`).

## Тесты

`src/test/java/.../roman/` (JUnit 4, только чистая логика):

- `RomanNumeralsTest` — ключевые значения `toRoman` (1, 4, 9, 14, 40,
  49, 90, 400, 900, 1000, 1994, 3999…), полный roundtrip
  `fromRoman(toRoman(n)) == n` для 1..3999, отказ на неканонических и
  некорректных записях.
- `RomanGameTest` — диапазоны и цели по сложностям, согласованность
  вопроса с направлением, уникальность вариантов и наличие верного,
  неповторение чисел подряд, детерминизм с одинаковым seed,
  прогресс/победа/ошибки, отказ на неверном индексе.

Запуск: `./gradlew :feature:games:roman:test`

## Запуск в демо

Игра добавлена в демо-приложение со всеми мини-играми (выбор сложности,
без будильника и Firebase):

```bash
./gradlew :feature:games:demo:installDebug
```

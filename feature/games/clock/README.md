# Игра «Который час» (`:feature:games:clock`)

Мини-игра будильника SmartAlarm (**game id 18**). На экране — аналоговый
циферблат со стрелками; игрок должен «прочитать» время и выбрать его цифровую
запись («07:45») из четырёх кнопок. Нужно верно ответить в нескольких раундах
подряд — тематически идеальная тренировка для человека, который только что
проснулся и смотрит на часы.

## Правила

1. В каждом раунде генерируется случайное время (часы 1..12, минуты кратны
   шагу сложности) и рисуется на циферблате.
2. Под циферблатом — четыре варианта ответа в формате «ЧЧ:ММ»; ровно один
   правильный.
3. Правильный ответ — переход к следующему раунду. Пройдены все раунды —
   победа, будильник выключается, показывается экран результата.
4. Ошибка — штраф −10 очков и **новый раунд с тем же номером** (время и
   варианты перегенерируются, чтобы ответ нельзя было подобрать перебором).
5. Системная «назад» в режиме пробы возвращает к выбору игр; когда игра
   запущена будильником — не делает ничего (от будильника так не сбежать).

## Сложности

| Сложность | Шаг минут | Раундов | Комментарий                              |
|-----------|-----------|---------|------------------------------------------|
| 1         | 30        | 3       | только «ровно» и «половина»              |
| 2         | 15        | 4       | + «четверть» и «без четверти»            |
| 3         | 5         | 5       | любое «настоящее» время, стрелки близко  |

Чем меньше шаг, тем ближе дистракторы к правильному ответу и тем внимательнее
нужно смотреть на положение часовой стрелки.

## Математика стрелок

Реализована в `ClockTime` (чистый Kotlin, без Android):

- **минутная стрелка**: `минуты × 6°` (полный круг 360° за 60 минут);
- **часовая стрелка**: `(часы mod 12) × 30° + минуты × 0.5°` — часовая
  стрелка смещается пропорционально минутам, как на настоящих часах
  (30° за час = 0.5° за минуту).

Примеры: 12:00 → 0°/0°; 3:00 → 90°/0°; 6:30 → 195°/180°; 9:45 → 292.5°/270°.

Углы отсчитываются по часовой стрелке от отметки «12». `ClockFaceView` сама
ничего не считает — получает готовые углы через `setHandAngles(...)`.

## Дистракторы

Кандидаты в порядке приоритета (`ClockGame.generateRound`), все валидные
времена (часы 1..12, минуты 0..59), уникальные и не равные правильному:

1. **Перепутанные стрелки** (`withSwappedHands`) — часовая встаёт туда, куда
   указывала минутная, и наоборот: 9:15 → 3:45. Классическая ошибка чтения.
2. **±шаг сложности** (`shiftedBy(±minuteStep)`) — соседние времена: для
   7:45 на сложности 3 это 7:40 и 7:50.
3. **Зеркальное время** (`mirrored`) — обе стрелки отражены относительно
   вертикальной оси «12–6»: 4:40 → 7:20.
4. Если похожих кандидатов не хватило (совпали с правильным или между собой,
   например у 12:00 «перепутанный» и «зеркальный» варианты равны ему самому) —
   добираются случайные времена с шагом 5 минут.

Итоговые 4 варианта перемешиваются. Вся случайность идёт через
`kotlin.random.Random`, передаваемый параметром, — с фиксированным seed партия
полностью воспроизводима (используется в тестах).

## Очки

Как в игре-образце «Арифметика» (`:feature:games:calc`):

- каждая ошибка: **−10** очков;
- победа: **+ (600 − прошло_секунд) × сложность** (бонус за скорость).

## Архитектура (файлы и роли)

```
src/main/java/com/example/smartalarm/feature/games/clock/
├── ClockTime.kt          — чистая модель времени: валидация (1..12 / 0..59),
│                           углы стрелок, формат «ЧЧ:ММ», shiftedBy /
│                           withSwappedHands / mirrored / fromTotalMinutes
├── ClockGame.kt          — чистая логика партии: параметры сложности, раунды
│                           (ClockRound = время + 4 варианта), answer(),
│                           генерация дистракторов; без Android-импортов
├── ClockFaceView.kt      — кастомная View циферблата: круг, 60 отметок,
│                           цифры 1..12, две стрелки; квадратная (onMeasure),
│                           цвета из атрибутов темы (colorPrimary /
│                           colorSecondary / colorOnSurface) с фолбэками
├── ClockGameViewModel.kt — AndroidViewModel: секундомер «м.сс», счёт, доступ
│                           к БД будильников и перезапуск будильника; держит
│                           ClockGame (партия переживает поворот экрана)
└── ClockGameFragment.kt  — экран игры: контракт будильника, навигация,
                            привязка ClockGame к UI
src/main/res/layout/
└── fragment_clock_game.xml — ConstraintLayout: таймер, «Который час?»,
                              счётчик раундов, ClockFaceView (1:1),
                              4 кнопки 2×2, FAB «Выключить музыку»
src/test/java/.../clock/
├── ClockTimeTest.kt      — углы, формат, арифметика, валидация
└── ClockGameTest.kt      — сложности, раунды, варианты, seed-детерминизм
```

Слои: `ClockTime`/`ClockGame` — чистый Kotlin (юнит-тестируются на JVM),
`ClockGameViewModel` — тонкая обёртка (таймер/очки/будильник),
`ClockGameFragment`/`ClockFaceView` — только отображение.

## Контракт экрана (общий для всех игр)

Аргументы фрагмента (ключи с пробелами — исторические): `alarm id` (Long),
`start time` (Long, 0 = проба), `test` (Boolean), `difficulty` (Int 1..3),
`music path` (String).

- `onResume`: если не `test` — снять уведомление будильника
  (`NotificationManagerCompat`, id = `alarm id`);
- `onPause`: если не `test` и игра не пройдена — `viewModel.startNewAlarm()`
  перезапускает будильник;
- каждые две минуты бездействия (чётная ненулевая минута таймера, секунды 00)
  заново включается мелодия (`AlarmMediaPlayer.playAudio`) и появляется кнопка
  «Выключить музыку» (`AlarmMediaPlayer.stopAudio()`);
- победа: в аргументы кладутся `time` (String, «м.сс»), `score` (Int),
  `game id` (Int = 18), затем переход к экрану результата — в пробном и
  будильничном режимах это разные action'ы, т.к. игра есть в обоих nav-графах
  (`action_clockGameFragment2_to_gameResultFragment2` /
  `action_clockGameFragment_to_gameResultFragment`).

## Тесты

```bash
./gradlew :feature:games:clock:test
```

JUnit4, только чистая логика (без Android): углы стрелок, формат строки,
кратность минут шагу сложности, уникальность/валидность/полнота вариантов,
детерминизм с одинаковым seed, продвижение раундов, победа, обработка ошибок.

## Запуск в демо

Игра добавлена в демо-приложение со всеми мини-играми (без будильника
и Firebase):

```bash
./gradlew :feature:games:demo:installDebug
```

На стартовом экране демо выбрать «Который час» и сложность.

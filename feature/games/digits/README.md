# Игра «Запомни число» (`:feature:games:digits`)

Мини-игра для будильника SmartAlarm, **game id = 19**. Чтобы выключить
будильник, нужно три раза подряд запомнить и по памяти ввести число.

## Правила

1. На экране в карточке показывается случайное число (строка цифр без
   ведущего нуля: первая цифра 1..9, остальные 0..9).
2. Через несколько секунд число скрывается — вместо него маска «•» той же
   длины, появляются поле ввода (`inputType=number`) и кнопка «Проверить».
3. Игрок вводит число по памяти. Ответ сравнивается **как строка**, поэтому
   пустой ввод и ввод с ведущими нулями («0123» вместо «123») — неверные.
4. Верный ответ — следующий раунд, число становится на одну цифру длиннее.
5. Неверный ответ — **−10 очков** и НОВОЕ число той же длины: оно снова
   показывается на полное время, раунд не меняется.
6. Всего **3 раунда**; верный ответ в третьем — победа, будильник выключен.

## Сложности

| Сложность | Длины чисел по раундам | Время показа |
|-----------|------------------------|--------------|
| 1         | 4 → 5 → 6 цифр         | 3.0 с        |
| 2         | 5 → 6 → 7 цифр         | 2.5 с        |
| 3         | 6 → 7 → 8 цифр         | 2.0 с        |

Некорректное значение сложности трактуется как 1.

## Очки

- Каждая ошибка: **−10** очков.
- Финиш (как в остальных играх, см. калькулятор):
  `score += (600 − прошло_секунд_с_начала) × difficulty`.

## Архитектура

Чистая логика отделена от Android и тестируется на JVM:

- `DigitsLogic.kt` — объект без Android-зависимостей: длины раундов и время
  показа по сложности, генерация числа-строки (случайность передаётся
  параметром `kotlin.random.Random` — для детерминизма в тестах), проверка
  ответа (посимвольное сравнение строк), признак последнего раунда,
  константа `TOTAL_ROUNDS = 3`.
- `DigitsGameEngine.kt` — чистая машина состояний партии: текущий раунд,
  загаданное число, `submitAnswer()` с результатом `AnswerResult`
  (`WIN` / `NEXT_ROUND` / `WRONG`), флаг победы. После победы состояние
  не меняется.
- `DigitsGameViewModel.kt` — `AndroidViewModel` по образцу
  `CalcGameViewModel`: владеет `DigitsGameEngine` (поэтому раунд, число и
  фаза переживают поворот экрана), хранит фазу (`DigitsPhase.SHOWING` /
  `INPUT`) в `LiveData`, момент начала показа `showPhaseStartMillis` и
  `remainingShowTimeMillis()` — после поворота в фазе показа число
  показывается только **остаток** времени. Также: игровой таймер («м.сс»),
  счёт, загрузка будильника из Room и его перезапуск
  (`AlarmDbRepository` / `AlarmCreateRepository`).
- `DigitsGameFragment.kt` — UI и тайминг: `Handler.postDelayed` планирует
  скрытие числа на остаток времени показа (сама логика о тайминге не знает);
  реализует контракт экрана игры (ниже). В фазе ввода содержимое `EditText`
  сохраняется системой при повороте.
- `res/layout/fragment_digits_game.xml` — `ConstraintLayout`: таймер, счётчик
  раундов, карточка с крупным числом/маской, карточка с полем ввода, кнопка
  «Проверить», кнопка выключения музыки. Цвета — через атрибуты темы
  (`?attr/colorPrimary` и т.д.), как в остальных играх.

## Контракт экрана игры

Аргументы фрагмента (ключи исторические, с пробелами): `alarm id` (Long),
`start time` (Long, 0 = проба), `test` (Boolean), `difficulty` (Int 1..3),
`music path` (String).

- `onResume`: если не `test` — снять уведомление будильника
  (`NotificationManagerCompat`, id = `alarm id`).
- `onPause`: если не `test` — `viewModel.startNewAlarm()` (перезапуск
  будильника, если игра не пройдена).
- Каждые две минуты бездействия (чётная ненулевая минута таймера, секунды 00)
  при не-`test` заново включается мелодия (`AlarmMediaPlayer.playAudio`) и
  появляется кнопка её выключения (`AlarmMediaPlayer.stopAudio()`).
- Системная «назад»: в `test` — возврат к выбору игр
  (`action_digitsGameFragment2_to_gameChoiceFragment`, popUpTo
  `alarmsFragment`); при будильнике — игнорируется (выйти нельзя).
- Победа: в аргументы кладутся `time` (String), `score` (Int),
  `game id` (Int = 19); переход к результату — в `test` через
  `action_digitsGameFragment2_to_gameResultFragment2`, при будильнике через
  `action_digitsGameFragment_to_gameResultFragment` +
  `viewModel.setPositiveResult()`.

Destination'ы в обоих nav-графах `:core:ui` и в демо уже заведены.

## Тесты

JVM-тесты (JUnit4) покрывают только чистую логику, `src/test/java/...`:

- `DigitsLogicTest` — длины раундов и время показа по сложностям (включая
  fallback), рост длины на 1 за раунд, длина/первая цифра ≠ 0/только цифры у
  сгенерированного числа, детерминизм с одинаковым seed, проверка ответа
  (верный, неверный, с ведущим нулём, пустой), признак последнего раунда.
- `DigitsGameEngineTest` — стартовое состояние, верный ответ двигает раунд и
  удлиняет число, ошибка НЕ двигает раунд и перегенерирует число той же
  длины, победа за 3 верных ответа (в том числе с ошибками по пути),
  прогрессия длин по сложностям, детерминизм движка с seed, неизменность
  состояния после победы.

Запуск: `./gradlew :feature:games:digits:test`.

## Запуск в демо

Игра доступна в демо-приложении со всеми мини-играми (без будильника и
Firebase):

```bash
./gradlew :feature:games:demo:installDebug
```

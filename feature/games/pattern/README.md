# Игра «Закономерность» (`:feature:games:pattern`)

Мини-игра будильника SmartAlarm, **game id = 24** (см. `ALL_GAMES` в `:core:data`).

## Правила

На экране — ряд эмодзи, построенный по повторяющемуся паттерну, и стрелка
с вопросом: `🔵 🔺 🔵 🔺 🔵 → ?`. Нужно понять закономерность и выбрать
следующий элемент ряда из четырёх вариантов. Верный выбор открывает
следующий раунд; после последнего раунда игра пройдена и будильник можно
выключить.

Ошибочный выбор штрафуется, и — главное — **раунд генерируется заново**
(новый паттерн, новые символы, новые варианты), поэтому перебор четырёх
кнопок не помогает: угадывать невыгодно, выгодно думать.

## Виды паттернов и сложности

Паттерн задаётся шаблоном из букв А/Б/В/Г — на место каждой буквы
подставляется свой эмодзи, при этом все буквы получают **разные** символы
(А≠Б≠В≠Г). Игроку показывается минимум **2 полных периода** шаблона плюс
случайные 0–2 элемента, ответ — элемент, следующий сразу за показанными.

| Сложность | Период | Шаблоны | Раундов |
|-----------|--------|---------|---------|
| 1 | 2 | АБ (АБАБАБ…) | 3 |
| 2 | 3 | АБВ, ААБ, АББ | 4 |
| 3 | 4 | АБВГ, ААББ, АБАВ | 5 |

Число раундов равно `difficulty + 2`; сложность вне 1..3 приводится к
границам диапазона. Один и тот же шаблон не выпадает два раунда подряд
(на сложности 1 шаблон один, поэтому там ограничение не действует).

## Пул эмодзи

Пул объявлен в `PatternLogic.kt` (`EMOJI_POOL`) — 16 визуально хорошо
различимых символов:

🔵 🔺 ⭐ 🍎 🌙 ❤️ 🍀 ⚡ 🐟 🎈 🍩 🔔 🌸 🐝 ⚽ 🎁

Из пула случайно берутся символы паттерна (2–4 штуки в зависимости от
шаблона) и посторонние варианты ответа.

## Варианты ответа

В каждом раунде ровно 4 уникальных варианта в случайном порядке:

1. правильный ответ;
2. до двух **других символов этого же паттерна** (самые коварные
   дистракторы — они есть в ряду, но не следующие);
3. **посторонние эмодзи** из пула, которых в паттерне нет, — ими набор
   дополняется до четырёх (посторонний вариант есть всегда, минимум один).

## Очки

Формула — по образцу арифметики (`:feature:games:calc`):

```
score = −10 × ошибки + (600 − прошло_секунд) × сложность
```

Каждый ошибочный выбор стоит 10 очков; чем быстрее пройдена игра и чем
выше сложность, тем больше итоговый счёт (`computePatternScore`).

## Архитектура

| Файл | Роль |
|------|------|
| `PatternLogic.kt` | Чистая логика без Android-зависимостей: пул `EMOJI_POOL`, шаблоны (`PatternTemplate`, `templatesFor`), раунд (`PatternRound` — ряд, ответ, варианты), игра (`PatternGame` — раунды, ошибки, победа), счёт (`computePatternScore`). Вся случайность — через параметр `kotlin.random.Random`, поэтому логика детерминированно тестируется с seed. |
| `PatternGameViewModel.kt` | `AndroidViewModel`, тонкий слой над `PatternGame`: таймер игры (строка «м.сс»), загрузка будильника из Room (`AlarmDbRepository`/`AlarmsDB`) и его перезапуск через `AlarmCreateRepository`, если игра не пройдена, подсчёт итогового счёта. |
| `PatternGameFragment.kt` | Экран игры: отрисовка раунда (ряд, счётчик «Раунд N из M», 4 кнопки-варианта), обработка контракта экрана игры (см. ниже). |
| `res/layout/fragment_pattern_game.xml` | Layout экрана (ConstraintLayout, ViewBinding `FragmentPatternGameBinding`, цвета через `?attr/...`). |

## Контракт экрана игры

Стандартный контракт из `.claude/rules/games.md` (образец — калькулятор):

- аргументы: `alarm id` (Long), `start time` (Long, 0 = проба), `test`
  (Boolean), `difficulty` (Int 1..3), `music path` (String);
- `onResume` — снятие уведомления будильника (кроме `test`);
- `onPause` — перезапуск будильника через `viewModel.startNewAlarm()`, если
  игра не пройдена (кроме `test`);
- каждые две минуты бездействия снова включается мелодия
  (`AlarmMediaPlayer.playAudio`) и появляется кнопка «Выключить музыку»
  (`AlarmMediaPlayer.stopAudio`);
- системная «назад»: в режиме `test` — возврат к выбору игр
  (`action_patternGameFragment2_to_gameChoiceFragment`), в режиме будильника
  — игнорируется;
- победа: в аргументы кладутся `time` (String), `score` (Int),
  `game id` (Int = 24) и происходит переход к результату
  (`action_patternGameFragment2_to_gameResultFragment2` в `test`,
  `action_patternGameFragment_to_gameResultFragment` — из будильника,
  с `viewModel.setPositiveResult()`).

Destination'ы объявлены в обоих nav-графах `:core:ui` (`nav_graph.xml`,
`nav_game_graph.xml`) и в графе демо-приложения.

## Тесты

`src/test/java/.../PatternGameTest.kt` — JUnit4, только чистая логика:

- пул: размер ≥ 12, уникальность;
- шаблоны: периоды 2/3/4 по сложностям, индексы букв без пропусков,
  уникальность, приведение сложности к 1..3;
- раунд: показанный ряд действительно следует шаблону (пересчёт по периоду
  в тесте), ответ — следующий элемент, длина ряда в диапазоне
  `2×период .. 2×период + 2`, символы паттерна различны и из пула;
- варианты: ровно 4, уникальны, содержат верный ответ и хотя бы один
  посторонний эмодзи;
- ошибки: счётчик растёт, раунд не продвигается, перебор вариантов не
  приводит к победе;
- победа после всех раундов (3/4/5), клики после победы игнорируются;
- шаблон не повторяется два раунда подряд (сложности 2–3);
- детерминизм: одинаковый seed — одинаковая игра;
- счёт: формула, цена ошибки, отрицательный счёт при долгой игре.

Запуск: `./gradlew :feature:games:pattern:test`.

## Запуск в демо

Игра включена в демо-приложение со всеми мини-играми (без будильника и
Firebase):

```bash
./gradlew :feature:games:demo:installDebug
```

# Игра «По порядку» (`:feature:games:sorting`)

Мини-игра для будильника SmartAlarm, **game id = 4** (см. `ALL_GAMES` в `:core:data`).

## Правила

На экране — сетка кнопок (3 столбца) со случайными **уникальными** числами.
Нужно нажимать числа **строго по возрастанию**: от наименьшего к наибольшему.

- Правильное нажатие — кнопка «гаснет» (становится неактивной и полупрозрачной)
  и выводится из игры.
- Ошибочное нажатие — показывается «Неправильно!» и снимается **10 очков**.
  Раунд при ошибке **не сбрасывается**: уже погашенные числа остаются погашенными
  (решение осознанное — игра выключает будильник, полусонному игроку сброс
  прогресса был бы слишком жесток; наказание только очками, как в «Арифметике»).
- Победа — когда нажаты все числа.

## Сложности

| Сложность | Чисел | Диапазон | Особенности |
|-----------|-------|----------|-------------|
| 1 | 6 (сетка 3×2) | 1..50 | — |
| 2 | 9 (сетка 3×3) | 1..199 | диапазон шире — числа до трёх цифр |
| 3 | 12 (сетка 3×4) | −99..199, без нуля | **ровно 3 отрицательных числа** (гарантированно) |

Числа генерируются без повторов: диапазон перемешивается и берутся первые N
(на сложности 3 — отдельно 3 из −99..−1 и 9 из 1..199, затем общая перетасовка).
Порядок кнопок на поле — случайный. Неизвестная сложность трактуется как 1.

## Очки

- Ошибка: **−10** очков за каждое неверное нажатие.
- Победа: **+ (600 − прошло_секунд) × сложность** (формула как в «Арифметике»);
  при долгой игре и множестве ошибок итог может быть отрицательным — это
  поведение общее для игр проекта.

## Архитектура (файлы и роли)

- `src/main/java/.../SortingGameLogic.kt` — **чистая логика без Android-зависимостей**:
  - `SortingNumbersGenerator` — генерация набора чисел по сложности;
    случайность передаётся параметром `kotlin.random.Random` (детерминизм в тестах);
  - `SortingGame` — состояние партии: какие числа погашены (`isPressed`),
    какое ждём следующим (`nextExpected`), счётчик ошибок (`mistakes`),
    обработка нажатия (`press` → `CORRECT` / `WRONG` / `WIN`);
    после победы `press` идемпотентен;
  - `PressResult` — результат нажатия.
- `src/main/java/.../SortingGameViewModel.kt` — `AndroidViewModel` по образцу
  `CalcGameViewModel`: секундный таймер (`timeCurrentString`, формат `м.сс`),
  очки, загрузка будильника (`AlarmDbRepository`) и его перезапуск
  (`AlarmCreateRepository`), если игра не пройдена. Партия (`SortingGame`)
  живёт во ViewModel и переживает поворот экрана (`startGame` идемпотентен).
- `src/main/java/.../SortingGameFragment.kt` — UI: строит кнопки программно
  в `android.widget.GridLayout` (3 столбца, равная ширина через weight)
  и реализует контракт экрана игры.
- `src/main/res/layout/fragment_sorting_game.xml` — `ConstraintLayout`
  (таймер, подсказка, карточка с `GridLayout`, кнопка выключения музыки);
  цвета — через атрибуты темы (`?attr/...`).

## Контракт экрана (см. `.claude/rules/games.md`)

Аргументы фрагмента: `alarm id` (Long), `start time` (Long, 0 = проба),
`test` (Boolean), `difficulty` (Int 1..3), `music path` (String).

- `onResume`: вне `test` — снимает уведомление будильника (id = `alarm id`).
- `onPause`: вне `test` — `viewModel.startNewAlarm()` (перезапуск будильника,
  если игра не пройдена).
- Каждые две минуты (чётная ненулевая минута, секунды 00) вне `test` —
  повторно включается мелодия (`AlarmMediaPlayer.playAudio`) и появляется
  кнопка «Выключить музыку» (`AlarmMediaPlayer.stopAudio()`).
- Системная «назад»: в `test` — возврат к выбору игр
  (`action_sortingGameFragment2_to_gameChoiceFragment`,
  `popUpTo(alarmsFragment, true)`); вне `test` — игнорируется.
- Победа: в аргументы кладутся `time` (String), `score` (Int),
  `game id` (Int = 4); переход к результату:
  - `test` → `action_sortingGameFragment2_to_gameResultFragment2`;
  - будильник → `action_sortingGameFragment_to_gameResultFragment`
    и `viewModel.setPositiveResult()`.

Destination'ы объявлены в `:core:ui` (`nav_graph.xml` — проба,
`nav_game_graph.xml` — будильник) и в демо-графе.

## Тесты

`src/test/java/.../SortingNumbersGeneratorTest.kt` и `SortingGameTest.kt`
(JUnit4, только чистая логика):

- количество, уникальность и диапазоны чисел для каждой сложности
  (по 100 seed'ов), ровно 3 отрицательных на сложности 3, отсутствие нуля;
- детерминизм: один seed → один набор; разные seed'ы → разные наборы;
- правильная последовательность нажатий до победы (включая сгенерированные
  наборы всех сложностей), `WIN` на последнем числе;
- ошибки: счётчик растёт, прогресс не сбрасывается, число вне поля — ошибка;
- идемпотентность `press` после победы; отклонение дубликатов и пустого списка.

Запуск: `./gradlew :feature:games:sorting:test`.

## Запуск в демо

Игра включена в демо-приложение со всеми мини-играми (выбор сложности,
без будильника и Firebase):

```bash
./gradlew :feature:games:demo:installDebug
```

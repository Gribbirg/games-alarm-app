# Игра «Анаграммы» (`:feature:games:anagram`)

Мини-игра для будильника SmartAlarm (game id = **10**): чтобы выключить
будильник, нужно собрать несколько русских слов из перемешанных букв.

## Правила

- На экране показываются кнопки с буквами загаданного слова в перемешанном
  порядке (порядок гарантированно отличается от исходного слова, если у слова
  есть хотя бы две разные буквы).
- Игрок нажимает буквы по одной — собранная часть слова отображается сверху
  в карточке, недобранные буквы показаны прочерками. Нажатая кнопка гаснет:
  учёт идёт по индексу кнопки, поэтому одинаковые буквы в слове (например,
  два «о» в «потоп») не путаются.
- Кнопка **«Сброс»** очищает набранное и снова включает все кнопки-буквы.
- Кнопка **«Проверить»**:
  - слово собрано не полностью — подсказка «Слово собрано не полностью!»,
    без штрафа;
  - слово неверное — «Неправильно!», штраф −10 очков, набор сбрасывается;
  - слово верное — «Верно!» и следующее слово (буквы перемешиваются заново);
  - собрано последнее слово — победа, переход на экран результата.
- Слова в рамках одной игры не повторяются.

## Сложность

| Уровень | Словарь | Длина слов | Слов до победы |
|---------|---------|------------|----------------|
| 1 | `AnagramWords.EASY` | 4–5 букв | 2 |
| 2 | `AnagramWords.MEDIUM` | 6–7 букв | 3 |
| 3 | `AnagramWords.HARD` | 8+ букв | 3 |

## Словари

Словари лежат прямо в коде логики (`logic/Words.kt`, объект `AnagramWords`).
Принципы:

- только существительные в именительном падеже единственного числа;
- повседневная лексика без редких и устаревших слов — спросонья слово должно
  узнаваться сразу;
- строчные буквы, «ё» не заменяется на «е» (в UI буквы показываются
  заглавными);
- длина строго соответствует уровню; в каждом списке ≥25 слов без дубликатов;
- у каждого слова минимум две разные буквы, иначе его нельзя перемешать
  «по-другому».

## Очки

Как в игре-образце «Калькулятор»:

- каждое неверно собранное слово: **−10** очков;
- за победу: **`(600 − прошло_секунд) × difficulty`** прибавляется к
  накопленному (при долгой игре итог может уйти в минус — так же, как в
  остальных играх).

## Архитектура

```
src/main/java/com/example/smartalarm/feature/games/anagram/
├── logic/Words.kt            — AnagramWords: словари по уровням, forDifficulty()
├── logic/AnagramLogic.kt     — чистая логика без Android:
│                               shuffleWord() (перемешивание ≠ исходному),
│                               enum SubmitResult, класс AnagramGame
│                               (выбор слов без повторов, набор по индексам,
│                               сброс, проверка, прогресс)
├── AnagramGameViewModel.kt   — AndroidViewModel: таймер («м.сс», раз в
│                               секунду), очки, доступ к будильнику
│                               (AlarmDbRepository/AlarmCreateRepository),
│                               владеет экземпляром AnagramGame (переживает
│                               пересоздание фрагмента)
└── AnagramGameFragment.kt    — UI: кнопки-буквы создаются программно рядами
                                (LinearLayout) по состоянию игры, навигация,
                                контракт экрана игры
src/main/res/layout/fragment_anagram_game.xml — разметка (ConstraintLayout)
src/test/java/...             — юнит-тесты чистой логики (JUnit4)
```

Случайность передаётся в логику параметром `kotlin.random.Random`, поэтому
поведение полностью детерминировано при фиксированном seed (используется в
тестах).

## Контракт экрана игры

Соблюдается общий контракт (`.claude/rules/games.md`), образец —
`CalcGameFragment`:

- аргументы: `alarm id` (Long), `start time` (Long, 0 = проба),
  `test` (Boolean), `difficulty` (Int 1..3), `music path` (String);
- `onResume`: если не `test` — снимается уведомление будильника
  (id = `alarm id`);
- `onPause`: если не `test` и игра не пройдена — будильник перезапускается
  (`viewModel.startNewAlarm()`);
- на каждой чётной ненулевой минуте (секунды 00) при не-`test` заново
  включается мелодия (`AlarmMediaPlayer.playAudio`) и появляется кнопка
  выключения музыки;
- системная «назад»: в `test` — возврат к выбору игр
  (`action_anagramGameFragment2_to_gameChoiceFragment`), при будильнике —
  игнорируется;
- победа: в аргументы кладутся `time` (String), `score` (Int),
  `game id` (Int = 10); в `test` — переход по
  `action_anagramGameFragment2_to_gameResultFragment2`, иначе —
  `action_anagramGameFragment_to_gameResultFragment` +
  `viewModel.setPositiveResult()`.

## Тесты

`./gradlew :feature:games:anagram:test` — JUnit4, только чистая логика:

- `AnagramWordsTest` — длины слов соответствуют уровням, ≥25 слов на уровень,
  нет дубликатов, только строчные русские буквы, у всех слов ≥2 разных букв;
- `AnagramGameTest` — перемешивание отличается от исходного слова для всех
  слов словарей и сохраняет мультимножество букв; детерминизм с seed; набор
  по индексам с буквами-дублями; запрет повторного нажатия той же кнопки;
  сброс; неполное/неверное/верное слово; отсутствие повторов слов за игру;
  победа и поведение после неё.

## Запуск в демо

Игра включена в демо-приложение со всеми мини-играми (без будильника и
Firebase):

```bash
./gradlew :feature:games:demo:installDebug
```

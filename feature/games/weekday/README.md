# Мини-игра «День недели» (`:feature:games:weekday`)

Игра ментальной календарной арифметики для будильника SmartAlarm
(game id = **25**, запись `GameData(25, "День недели")` в `ALL_GAMES`).

## Правила

На экране показывается вопрос вида:

> Сегодня — среда. Какой день недели будет через 9 дней?

или про прошлое:

> Сегодня — понедельник. Какой день недели был 4 дня назад?

Под вопросом — **4 кнопки** с уникальными днями недели, среди которых ровно
один верный. Игрок отвечает нажатием кнопки:

- **верный ответ** — счётчик прогресса увеличивается, показывается следующий
  вопрос;
- **ошибка** — минус 10 очков, счётчик прогресса не меняется, показывается
  **новый** вопрос (пересдать тот же нельзя).

Побеждает игрок, давший нужное число верных ответов (см. сложности).
Направление смещения (в будущее/в прошлое) выбирается равновероятно.

## Сложности

| Сложность | Верных ответов для победы | Модуль смещения, дней | Формулировки |
|-----------|---------------------------|-----------------------|--------------|
| 1         | 3                         | 1..7                  | только дни |
| 2         | 4                         | 5..30                 | только дни |
| 3         | 5                         | 20..100               | дни, а в ~половине вопросов — «через 3 недели и 4 дня» / «3 недели и 4 дня назад» |

Числительные склоняются корректно («1 день», «2 дня», «5 дней»,
«1 неделю», «2 недели», «5 недель»); если дней в недельной записи 0,
остаётся только «N недель».

## Очки

Как в игре-образце «Калькулятор» (`:feature:games:calc`):

- каждая ошибка: **−10** очков;
- при победе: **+ (600 − прошло_секунд) × сложность** (может уйти в минус,
  если решать дольше 10 минут).

## Архитектура

Чистая логика отделена от Android и полностью покрыта JVM-тестами:

| Файл | Роль |
|------|------|
| `src/main/java/.../Weekday.kt` | Enum дней недели с русскими названиями и `shifted(offset)` — арифметика по модулю 7, корректная для отрицательных и больших смещений. Без Android-импортов. |
| `src/main/java/.../WeekdayQuestion.kt` | Модель вопроса `WeekdayQuestion` и `WeekdayQuestionFactory`: диапазоны смещений по сложности, генерация вопроса, текст (включая склонения и недельную запись), 4 уникальных варианта. Случайность — через параметр `kotlin.random.Random` (детерминизм с seed). Без Android-импортов. |
| `src/main/java/.../WeekdayGame.kt` | Состояние партии: счётчики верных ответов/ошибок, `answer()`, признак победы `isWon`, число вопросов для победы по сложности. Без Android-импортов. |
| `src/main/java/.../WeekdayGameViewModel.kt` | `AndroidViewModel`: таймер «м.сс», счёт, доступ к Room (`AlarmDbRepository`), перезапуск будильника (`AlarmCreateRepository`), LiveData вопроса/вариантов/прогресса. |
| `src/main/java/.../WeekdayGameFragment.kt` | Экран игры: контракт экрана (см. ниже), навигация, кнопки ответов. |
| `src/main/res/layout/fragment_weekday_game.xml` | ConstraintLayout: таймер, прогресс, карточка вопроса, 4 кнопки, кнопка «Выключить музыку». Цвета — через `?attr/...`. |

Даты специально **не** используют `java.time`/`Calendar` — только модульная
арифметика по `ordinal` enum'а.

## Контракт экрана (см. `.claude/rules/games.md`)

Аргументы фрагмента: `alarm id` (Long), `start time` (Long, 0 — проба),
`test` (Boolean), `difficulty` (Int 1..3), `music path` (String).

- `onResume`: если не `test` — снять уведомление будильника
  (`NotificationManagerCompat`, id = `alarm id`).
- `onPause`: если не `test` — `viewModel.startNewAlarm()` (будильник
  перезапускается, пока игра не пройдена).
- Каждые две минуты бездействия (чётная ненулевая минута таймера, секунды 00,
  не `test`) — заново включается мелодия (`AlarmMediaPlayer.playAudio`)
  и появляется кнопка её выключения (`AlarmMediaPlayer.stopAudio()`).
- Системная «назад»: в `test` — возврат к выбору игр
  (`action_weekdayGameFragment2_to_gameChoiceFragment`,
  `popUpTo(alarmsFragment, true)`); в режиме будильника — игнорируется
  (от будильника нельзя уйти кнопкой «назад»).
- Победа: в аргументы кладутся `time` (String), `score` (Int),
  `game id` = 25; в `test` — переход
  `action_weekdayGameFragment2_to_gameResultFragment2`, иначе
  `action_weekdayGameFragment_to_gameResultFragment` +
  `viewModel.setPositiveResult()`.

Destination'ы объявлены в обоих nav-графах `:core:ui`
(`nav_graph.xml` — `weekdayGameFragment2`, `nav_game_graph.xml` —
`weekdayGameFragment`) и в демо-графе.

## Тесты

`src/test/java/.../WeekdayLogicTest.kt` и `WeekdayGameTest.kt` (JUnit4,
только чистая логика):

- `shifted`: среда+9=пятница, понедельник−4=четверг, ±7k — тот же день,
  большие и отрицательные смещения, обратимость `shifted(o).shifted(-o)`;
- диапазоны смещений по сложностям (по 200 seed'ов на сложность);
- ответ согласован с текстом вопроса — тест независимо разбирает текст
  (день, числа, направление) и пересчитывает ответ;
- варианты: ровно 4, уникальны, содержат верный, верный встречается на всех
  позициях;
- склонения «день/дня/дней», «неделю/недели/недель», недельная запись;
- детерминизм генерации и целой партии при одинаковом seed;
- партия: победа после 3/4/5 верных ответов, ошибки не сбрасывают прогресс,
  после победы ответы игнорируются.

Запуск: `./gradlew :feature:games:weekday:test`.

## Запуск в демо

Игра доступна в демо-приложении со всеми мини-играми (без будильника
и Firebase):

```bash
./gradlew :feature:games:demo:installDebug
```

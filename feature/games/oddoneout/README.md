# Игра «Найди лишнее» (`:feature:games:oddoneout`)

Мини-игра будильника SmartAlarm, **game id = 8** (см. `ALL_GAMES` в `:core:data`).

## Правила

На экране — квадратная сетка одинаковых эмодзи, среди которых ровно один
отличается, причём он **визуально похож** на остальные (пары вроде 😺/😸 или
⭐/🌟). Нужно нажать на лишний символ. Верное нажатие открывает следующий
раунд, ошибочное — штрафуется и оставляет текущий раунд на месте. После
последнего раунда игра пройдена, будильник можно выключить.

## Сложности

| Сложность | Сетка | Раундов | Пары |
|-----------|-------|---------|------|
| 1 | 3×3 | 3 | заметные отличия (уровень похожести 1) |
| 2 | 4×4 | 4 | средняя похожесть (уровень 2) |
| 3 | 5×5 | 5 | самые похожие пары (уровень 3) |

Сторона сетки и число раундов равны `difficulty + 2`. Значение сложности
вне 1..3 приводится к границам диапазона.

## Пул пар эмодзи

Пул объявлен в `EmojiPairs.kt` (`EMOJI_PAIRS`), 15 пар — по 5 на каждый
уровень похожести. На сложности N используются только пары с
`similarity == N`. Одна и та же пара не выпадает два раунда подряд; какой из
двух символов пары «основной», а какой «лишний» — выбирается случайно.

Уровень 1 — заметные отличия:

| Пара | Комментарий |
|------|-------------|
| 🌕 / 🌝 | полная луна и луна с лицом |
| 🐨 / 🐼 | коала и панда |
| 🙈 / 🙉 | обезьянка закрыла глаза / уши |
| 👏 / 🙌 | аплодисменты и поднятые руки |
| 🦁 / 🐯 | лев и тигр |

Уровень 2 — средняя похожесть:

| Пара | Комментарий |
|------|-------------|
| 😺 / 😸 | улыбающийся кот и смеющийся кот |
| ⭐ / 🌟 | звезда и сияющая звезда |
| 🌸 / 💮 | сакура и белый цветок |
| 😴 / 😪 | спящий и сонный |
| 🐭 / 🐹 | мышь и хомяк |

Уровень 3 — самые похожие пары:

| Пара | Комментарий |
|------|-------------|
| 😀 / 😃 | улыбки, отличаются только глазами |
| 😄 / 😁 | смеющиеся лица |
| 😮 / 😯 | удивлённые лица |
| 😑 / 😐 | лица без эмоций (глаза-чёрточки и точки) |
| 🌛 / 🌜 | зеркальные месяцы с лицом |

Чтобы добавить пару, достаточно дописать `EmojiPair(a, b, similarity)` в
`EMOJI_PAIRS` — логика и тесты подстроятся автоматически (в каждом уровне
должно оставаться не меньше 2 пар).

## Очки

Формула по образцу арифметики (`:feature:games:calc`):

```
итог = −10 × ошибки + (600 − прошло_секунд) × сложность
```

- каждое ошибочное нажатие: −10 очков;
- на финише: + (600 − длительность игры в секундах) × difficulty.

Чистая функция — `computeOddOneOutScore()` в `OddOneOutGame.kt`.

## Архитектура

| Файл | Роль |
|------|------|
| `EmojiPairs.kt` | Чистая логика: модель `EmojiPair` и пул `EMOJI_PAIRS`. Без зависимостей от Android. |
| `OddOneOutGame.kt` | Чистая логика: `OddOneOutRound` (сетка раунда, индекс лишнего), `OddOneOutGame` (раунды, ошибки, победа, отсутствие повтора пар), `ClickResult`, `computeOddOneOutScore()`. Случайность инжектируется через `kotlin.random.Random`. |
| `OddOneOutGameViewModel.kt` | `AndroidViewModel`: таймер игры (строка «м.сс»), загрузка будильника (`AlarmDbRepository`/`AlarmsDB`) и его перезапуск (`AlarmCreateRepository`), тонкие делегаты к `OddOneOutGame`, итоговый счёт. |
| `OddOneOutGameFragment.kt` | UI: строит сетку эмодзи программно в `android.widget.GridLayout` (ячейки — `TextView` с весами), обрабатывает нажатия, контракт экрана игры. |
| `res/layout/fragment_oddoneout_game.xml` | Разметка: таймер, счётчик раундов, карточка с `GridLayout`, кнопка выключения музыки. Цвета — через `?attr/...` темы. |

## Контракт экрана (см. `.claude/rules/games.md`)

Аргументы фрагмента: `alarm id` (Long), `start time` (Long, 0 = проба),
`test` (Boolean), `difficulty` (Int 1..3), `music path` (String).

- `onResume`: если не `test` — снимается уведомление будильника;
- `onPause`: если не `test` и игра не пройдена — будильник перезапускается;
- каждые две минуты бездействия заново включается мелодия
  (`AlarmMediaPlayer`) и появляется кнопка её выключения;
- системная «назад»: в режиме `test` — возврат к выбору игр
  (`action_oddoneoutGameFragment2_to_gameChoiceFragment`), при реальном
  будильнике — игнорируется;
- при победе в аргументы кладутся `time` (String), `score` (Int),
  `game id` = 8 и происходит переход к результату
  (`action_oddoneoutGameFragment2_to_gameResultFragment2` в пробе,
  `action_oddoneoutGameFragment_to_gameResultFragment` при будильнике).

Destination'ы объявлены в обоих nav-графах `:core:ui` и в демо-графе;
ветки для id 8 есть в `LoadGameFragment` и `GameChoiceFragment`
(`:feature:games`).

## Тесты

`src/test/java/.../OddOneOutGameTest.kt` (JUnit4, только чистая логика):

- пул: ≥10 пар, ≥2 пары на каждый уровень похожести, символы в паре
  различны, пары уникальны;
- сложности: размеры сеток 3×3/4×4/5×5, число раундов 3/4/5, приведение
  сложности к 1..3, соответствие уровня похожести пар сложности;
- раунд: ровно один лишний символ и именно на заявленном индексе, символы
  раунда — из его пары;
- детерминизм: одинаковый seed даёт одинаковую игру;
- нет повтора пары два раунда подряд;
- нажатия: верное — следующий раунд, ошибочное — счётчик ошибок без смены
  раунда, победа после всех раундов, нажатия после победы игнорируются;
- счёт: формула, −10 за ошибку, возможность отрицательного счёта.

Запуск: `./gradlew :feature:games:oddoneout:test`.

## Запуск в демо

Игра включена в демо-приложение со всеми мини-играми (без будильника и
Firebase):

```bash
./gradlew :feature:games:demo:installDebug
```

В лаунчере демо выберите «Найди лишнее» и сложность.

# Игра «Цепочка» (`:feature:games:chain`)

Мини-игра устного счёта для будильника SmartAlarm (game id = **16** в `ALL_GAMES`).

## Правила

1. На экране появляется **стартовое число** (например, `17`).
2. По кнопке **«Дальше»** операции показываются по одной: `+ 7`, `× 2`, `− 5`…
   Показ по кнопке (а не по таймеру) выбран сознательно: только что
   проснувшийся игрок сам управляет темпом и не может «проспать» шаг.
3. Игрок считает в уме. После последней операции ещё одно нажатие «Дальше»
   показывает `= ?` — появляется поле ввода (только цифры) и кнопка
   **«Проверить»**.
4. **Верный ответ** — цепочка засчитана. На сложностях 1–2 этого достаточно
   для победы; на сложности 3 нужно решить **две** цепочки (см. ниже).
5. **Ошибка** — минус **10 очков** и **новая цепочка** с начала (стартовое
   число новое, уже решённые цепочки не сгорают).
6. **Пустой или нечисловой ввод** — не считается ответом: без штрафа и без
   смены цепочки, показывается подсказка «Введите итог».

## Сложности

| Сложность | Операций | Старт  | Операнд «+»/«−» | Множитель «×»       | Цепочек для победы |
|-----------|----------|--------|-----------------|---------------------|--------------------|
| 1         | 3        | 1..20  | 2..9            | нет                 | 1                  |
| 2         | 5        | 10..50 | 2..30           | 2..5 (шанс ~25%)    | 1                  |
| 3         | 7        | 10..99 | 10..99          | 2..9 (шанс ~30%)    | 2                  |

**Почему две цепочки на сложности 3:** длинная цепочка из 7 операций с большими
числами сама по себе требует концентрации, но одну цепочку всё же можно угадать
«на удаче»; вторая гарантирует, что игрок действительно проснулся, и при этом
не растягивает игру чрезмерно. Ошибка не отнимает уже решённые цепочки.

## Инварианты генерации (`ChainGenerator`)

- Стартовое число и **все промежуточные результаты** лежат в диапазоне
  **0..999**. Инвариант обеспечивается по построению, без отбрасывания
  неудачных цепочек:
  - `+` — операнд обрезается сверху так, чтобы сумма не превысила 999;
  - `−` — операнд обрезается сверху текущим значением (разность ≥ 0);
  - `×` — множитель обрезается сверху так, чтобы произведение не превысило
    999; умножение предлагается только при текущем значении от 2
    (умножать 0 и 1 тривиально) и только если минимальный множитель не
    выводит за 999.
- Хотя бы одна из операций `+`/`−` допустима при любом значении 0..999
  (их минимальный операнд ≤ 99), поэтому генерация шага всегда завершается.
- Вся случайность идёт через параметр `kotlin.random.Random`: с фиксированным
  seed генерация полностью детерминирована (используется в тестах).

## Очки

- Каждый неверный ответ: **−10**.
- При победе добавляется **(600 − прошедшие_секунды) × сложность** — та же
  формула, что и в остальных играх (образец — «Сосчитай»/calc).

## Архитектура

Чистая логика (без `import android.*`/`androidx.*`, покрыта юнит-тестами):

- `ChainOp.kt` — enum операции (`+`, `−`, `×`) с применением к значению;
- `ChainStep.kt` — шаг цепочки `ChainStep(op, operand)`, текст шага («+ 7»);
- `Chain.kt` — цепочка: старт, шаги, промежуточные результаты и итог;
- `ChainGenerator.kt` — генерация цепочек с инвариантом 0..999 (параметры
  сложностей — в таблице выше);
- `ChainGame.kt` — состояние партии: показ шагов (`showNextStep`), фаза ввода
  (`isInputPhase`), проверка ответа (`submitAnswer` →
  `EMPTY`/`WRONG`/`CHAIN_SOLVED`/`WIN`), счётчик решённых цепочек.

Android-слой:

- `ChainGameViewModel.kt` — `AndroidViewModel`, тонкий слой над `ChainGame`:
  таймер партии («м.сс»), очки, перезапуск будильника при выходе
  (`AlarmDbRepository`/`AlarmCreateRepository`), LiveData `displayText` /
  `progressText` / `inputPhase`. Партия живёт во ViewModel и переживает
  поворот экрана;
- `ChainGameFragment.kt` — экран игры (см. контракт ниже);
- `res/layout/fragment_chain_game.xml` — ConstraintLayout: таймер, крупный
  текст текущего шага в карточке, кнопка «Дальше», поле ввода итога с
  кнопкой «Проверить» (показываются в фазе ввода), кнопка выключения музыки.

## Контракт экрана игры

Реализован по общим правилам (`.claude/rules/games.md`, образец —
`CalcGameFragment`/`EquationGameFragment`):

- аргументы: `alarm id` (Long), `start time` (Long, 0 = проба), `test`
  (Boolean), `difficulty` (Int 1..3), `music path` (String);
- `onResume`: если не `test` — снимает уведомление будильника
  (`NotificationManagerCompat`, id = `alarm id`);
- `onPause`: если не `test` — `viewModel.startNewAlarm()` (будильник
  перезапускается, пока игра не пройдена);
- на каждой чётной ненулевой минуте (секунды 00) и не в `test` — заново
  включается мелодия (`AlarmMediaPlayer.playAudio`) и появляется кнопка
  выключения музыки;
- системная «назад»: в `test` — возврат к выбору игр
  (`action_chainGameFragment2_to_gameChoiceFragment`), не в `test` — ничего;
- победа: в аргументы кладутся `time` (String), `score` (Int),
  `game id` (Int = 16), переход к результату
  (`action_chainGameFragment2_to_gameResultFragment2` в `test`,
  `action_chainGameFragment_to_gameResultFragment` — от будильника)
  и `viewModel.setPositiveResult()`.

## Тесты

`src/test/java/com/example/smartalarm/feature/games/chain/` (JUnit4, только
чистая логика):

- `ChainGeneratorTest` — длины цепочек по сложностям (3/5/7); для сотен
  цепочек с фиксированным seed: старт и все промежуточные в 0..999, итог
  равен последовательному применению шагов (независимая реализация в тесте),
  операнды в диапазонах сложности, отсутствие умножения на сложности 1 и его
  наличие на 2–3, детерминизм seed, приведение сложности к 1..3;
- `ChainGameTest` — последовательность показа (старт → шаги → `= ?`),
  1 цепочка для победы на сложностях 1–2 и 2 на сложности 3, неверный ответ
  (новая цепочка, решённые не сгорают), пустой/нечисловой ввод (без смены
  состояния), ответ до фазы ввода игнорируется, поведение после победы,
  детерминизм seed.

Запуск: `./gradlew :feature:games:chain:test`.

## Запуск в демо

Игра добавлена в демо-приложение со всеми мини-играми (без будильника
и Firebase):

```bash
./gradlew :feature:games:demo:installDebug
```

На экране запуска выбрать «Цепочка» и сложность.

# :feature:games:memory — игра «Повтори узор»

Мини-игра будильника SmartAlarm (id игры в `ALL_GAMES` — **2**).
Пакет: `com.example.smartalarm.feature.games.memory`.

## Описание и правила

На экране — поле 3×3 из ячеек. Игра идёт раундами:

1. Приложение по очереди подсвечивает ячейки — показывает последовательность
   (в это время нажатия игрока игнорируются, статус «Смотрите...»).
2. Игрок повторяет последовательность нажатиями в том же порядке
   (статус «Повторите!»).
3. Если вся последовательность повторена верно — раунд пройден, к ней
   добавляется одна случайная ячейка, и показ повторяется (классический
   «Саймон»). Последовательность каждого следующего раунда — это
   последовательность предыдущего плюс одна новая ячейка.
4. Ошибка (нажата не та ячейка) — штраф −10 очков, прогресс ввода
   сбрасывается, и **та же** последовательность показывается заново.
5. Победа — верно повторена последовательность последнего раунда. После
   победы игрок попадает на экран результата, будильник считается выключенным.

Две одинаковые ячейки никогда не идут в последовательности подряд — двойную
подсветку одной ячейки легко не заметить.

## Сложность

Уровень сложности (аргумент `difficulty`, 1..3; значения вне диапазона
приводятся к ближайшей границе) задаёт стартовую длину, число раундов и
скорость показа (`MemoryGameSettings.forDifficulty`):

| Сложность | Старт | Раундов | Финальная длина | Подсветка, мс | Пауза, мс |
|-----------|-------|---------|-----------------|---------------|-----------|
| 1         | 3     | 3       | 5               | 650           | 300       |
| 2         | 4     | 4       | 7               | 500           | 250       |
| 3         | 5     | 5       | 9               | 400           | 200       |

## Очки

По образцу игры «Арифметика» (`:feature:games:calc`):

- каждая ошибка: **−10** очков (`MemoryGameLogic.MISTAKE_PENALTY`);
- при победе: **+ (600 − прошло_секунд) × сложность**.

Итог: `finalScore = −10 × ошибок + (600 − секунд_с_начала) × difficulty`
(`MemoryGameLogic.finalScore`). Счёт может быть отрицательным, если игра
длилась дольше 10 минут.

## Архитектура модуля

| Файл | Роль |
|------|------|
| `MemoryGameLogic.kt` | Чистая логика без Android-зависимостей: генерация последовательности, пошаговая проверка ввода (`onCellTapped` → `TapResult`), переход раундов (`startNextRound`), подсчёт ошибок и очков. Случайность — через параметр `kotlin.random.Random`, поэтому с seed поведение детерминировано (важно для тестов). |
| `MemoryGameSettings.kt` | Параметры сложности (таблица выше), чистый Kotlin. |
| `MemoryGameViewModel.kt` | Тонкий слой над логикой: хранит `MemoryGameLogic` (переживает поворот экрана — игра продолжается с текущего раунда), секундный таймер «м.сс», загрузка/перезапуск будильника (`AlarmDbRepository`, `AlarmCreateRepository`), итоговый счёт. |
| `MemoryGameFragment.kt` | Экран игры: подсветка последовательности (`Handler`-задержки, цвета из атрибутов темы `colorPrimaryContainer`/`colorPrimary`/`colorError`), обработка нажатий, контракт экрана игры и навигация. |
| `res/layout/fragment_memory_game.xml` | Разметка: таймер, номер раунда, статус, поле 3×3 из `MaterialCardView`, кнопка выключения музыки. |

## Контракт экрана (см. `.claude/rules/games.md`)

Аргументы фрагмента (ключи с пробелами — исторические):

- `alarm id` (Long), `start time` (Long, 0 = проба), `test` (Boolean),
  `difficulty` (Int 1..3), `music path` (String).

Поведение:

- `onResume`: если не `test` — снимается уведомление будильника
  (id уведомления = `alarm id`);
- `onPause`: если не `test` — будильник перезапускается, когда игра
  не пройдена (`viewModel.startNewAlarm()`);
- каждые две минуты бездействия (чётная ненулевая минута таймера, секунды 00)
  заново включается мелодия (`AlarmMediaPlayer.playAudio`) и появляется кнопка
  «Выключить музыку»;
- системная «назад»: в режиме пробы (`test == true`) — возврат к выбору игр
  (`action_memoryGameFragment2_to_gameChoiceFragment`); в режиме будильника —
  ничего не делает (в `nav_game_graph` экрана выбора игр нет);
- победа: в аргументы кладутся `time` (String, значение таймера),
  `score` (Int), `game id` (Int = 2), затем переход к результату:
  - проба: `action_memoryGameFragment2_to_gameResultFragment2` (`nav_graph`),
  - будильник: `action_memoryGameFragment_to_gameResultFragment`
    (`nav_game_graph`) плюс `viewModel.setPositiveResult()`.

Destination'ы объявлены в обоих nav-графах `:core:ui` и в графе демо;
модуль подключён в `settings.gradle.kts`, `app/build.gradle.kts` и
`feature/games/demo/build.gradle.kts`, ветки id = 2 есть в
`LoadGameFragment` и `GameChoiceFragment` хаба `:feature:games`.

## Тесты

JVM-тесты (JUnit 4, только чистая логика, без Android) в
`src/test/java/com/example/smartalarm/feature/games/memory/`:

- `MemoryGameSettingsTest` — параметры всех трёх сложностей и приведение
  значений вне диапазона;
- `MemoryGameLogicTest` — длина стартовой последовательности, диапазон ячеек
  0..8, отсутствие соседних дублей, детерминизм с одинаковым seed, пошаговая
  проверка ввода (верно/ошибка/раунд завершён), сброс прогресса и штраф при
  ошибке, рост последовательности с сохранением префикса, победа на последнем
  раунде, запрет лишнего `startNextRound`, формула очков.

Запуск:

```bash
./gradlew :feature:games:memory:test
```

## Запуск в демо-приложении

Игру можно запустить без будильника и Firebase из демо-приложения
(в основной APK оно не входит):

```bash
./gradlew :feature:games:demo:installDebug
```

В демо выбирается сложность и запускается «Повтори узор» одной кнопкой
(режим пробы: уведомления, перезапуск будильника и музыка не задействованы).
